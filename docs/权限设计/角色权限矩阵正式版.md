# 数据交易平台角色权限矩阵正式版

## 1. 文档说明

本矩阵基于：

- [全量用户角色说明.md](../用户角色说明/全量用户角色说明.md)
- [权限点清单.md](./权限点清单.md)

说明：

- 本矩阵只保留正式冻结的核心角色/身份。
- 页面、专题、阶段差异优先通过“权限包 + 作用域”表达，不再膨胀出大量岗位型角色。
- 矩阵中的 `R` 表示只读，`RW` 表示可读写/执行，`RV` 表示审核/复核，`-` 表示默认无权。
- 表中出现的“权限包”是对核心角色的附加授权，不构成新的正式角色。

## 2. 核心角色总览

### 2.1 平台人工角色

| 角色 | 定位 |
|---|---|
| 平台管理员 | 平台配置、高危治理、系统级管理 |
| 平台审核员 | 主体、商品、模板、合规统一审核 |
| 平台风控结算员 | 风险、冻结、争议、退款、赔付、结算执行 |
| 平台审计安全员 | 审计、安全、回放、证据、可观测性联查 |

### 2.2 租户人工角色

| 角色 | 定位 |
|---|---|
| 租户管理员 | 租户内最高业务管理角色 |
| 卖方运营员 | 卖方供给、交付、收款与接入执行角色 |
| 买方运营员 | 买方下单、验收、使用、退款申请执行角色 |
| 租户开发者 | 应用接入、SDK/API 调试、开发者通道 |
| 租户审计只读员 | 租户内只读审计与核对 |

### 2.3 服务与外部身份

| 身份 | 定位 |
|---|---|
| 租户应用身份 | 租户程序化调用身份 |
| 平台服务身份 | 平台内部服务、适配器、worker 身份 |
| 监管/合作方只读身份 | 外部只读/有限协同身份 |

## 5. 服务与外部身份矩阵

| 身份 | API 调用 | 交付/执行 | 审计/联查 | 跨链/连接器 | 安全边界 |
|---|---|---|---|---|---|
| 租户应用身份 | RW(授权范围内) | RW(API/模板/下载换票) | R(自身调用轨迹) | - | 受 SKU、合同、策略、配额约束 |

## 6. 权限包映射规则

为避免角色爆炸，以下能力改用权限包挂到核心角色上：

| 权限包 | 挂接角色 | 说明 |
|---|---|---|
| 供给治理权限包 | 租户管理员、卖方运营员 | 原“数据治理管理员、产品经理/上架管理员、卖方存储运营员”职责 |
| 买方交易权限包 | 租户管理员、买方运营员 | 原“采购经理、业务分析师、报告接收者”职责 |
| 法务/财务权限包 | 租户管理员 | 原“法务经理、财务经理”职责 |
| 搜索/推荐运营权限包 | 平台管理员 | 原“搜索运营员、推荐运营员”职责 |
| 一致性/运维权限包 | 平台审计安全员、平台管理员 | 原“一致性运营员、节点运维管理员、可观测性运营员、公链展示管理员”职责 |
| 受控执行权限包 | 平台管理员、平台服务身份、租户管理员、租户开发者 | 原“Sandbox Operator、Compute Environment Operator”职责 |
| 分润/跨链/生态权限包 | 平台风控结算员、平台管理员、平台服务身份 | 原“分润管理员、跨链管理员、连接器管理员、合作伙伴管理员”等职责 |
| 监管协同权限包 | 平台风控结算员、平台审计安全员、监管/合作方只读身份 | 原“监管协同管理员、监管协同操作员、审计只读监管角色”职责 |

## 8. V2/V3 扩展权限映射原则

`V2 / V3` 不再引入新核心角色，按以下规则扩展：

- 受控执行、模型、联邦、证明：
  - 平台审核员负责审核
  - 平台管理员负责环境和系统级治理
  - 平台服务身份负责执行
  - 租户管理员/租户开发者负责租户侧接入与任务发起
- 分润、跨链、生态、图风控：
  - 平台风控结算员负责业务执行和处置
  - 平台管理员负责系统级治理和配置
  - 平台审计安全员负责联查与审计
  - 监管/合作方只读身份仅在批准范围内只读或有限协同

## 9. 职责分离规则

以下动作不应落到同一人工账号：

- 商品提交 + 商品审核
- 合同模板绑定 + 合规审核
- 退款执行 + 争议裁决
- 结算执行 + 审计原文导出
- 系统高危配置 + break-glass 执行
- 冻结执行 + 解冻执行 + 证据删除

## 10. 菜单与接口解释规则

- 菜单、页面、接口中的“典型角色”必须优先引用本文件中的核心角色。
- 若某页面或接口需要表达专题能力，应写成“核心角色（某权限包）”，而不是重新造角色名。
- 例如：
  - `平台管理员（搜索/推荐运营权限包）`
  - `平台审计安全员（一致性/可观测性权限包）`
  - `租户管理员（法务/财务权限包）`

## 11. 一句话结论

角色权限矩阵的冻结结论是：

- 平台正式角色收敛为少量核心人工角色、服务身份和外部只读身份。
- 旧文件中的专题角色、岗位角色和未来角色，不再作为正式核心角色存在。
- 后续页面、菜单、接口和审计规则，只能在本矩阵定义的核心角色骨架上扩展权限包。
