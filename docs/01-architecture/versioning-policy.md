# 版本关联策略（BOOT-019）

本策略定义迁移、OpenAPI、Provider、演示数据、模板五类版本的关联关系，避免联调时口径漂移。

## 版本对象

- DB Migration 版本：`db/migrations/v1/*`
- OpenAPI 版本：`packages/openapi/*.yaml` 中的 `v1`
- Provider 版本：外部适配层实现版本（`mock`/`real`）
- Demo 数据版本：`fixtures/local/*`
- 模板版本：商品/契约/授权模板元数据版本

## 关联规则

1. Migration 变更必须触发 OpenAPI/模板兼容性检查；不兼容则提升次版本并写迁移说明。
2. OpenAPI 破坏性变更不得在 `v1` 内直接覆盖，必须新增兼容策略或明确阻断发布。
3. Provider 接口字段变化必须同步更新适配层契约测试与日志字段映射。
4. Demo 数据版本必须标注其依赖的 Migration 版本与模板版本。
5. 模板版本升级不得隐式改变已签约订单语义；订单使用创建时快照版本。

## 变更记录要求

- 每次版本变更必须在实施日志批次中记录影响面与回滚方式。
- 若仅文档更新且不改行为，标记为“文档版本更新”，不得混入行为变更。
