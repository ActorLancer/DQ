# 任务所有权矩阵（BOOT-016）

本矩阵用于区分“主控任务”和“可分发任务”，避免多 Agent 并行时边界漂移。

## 主控任务（必须由主控批次统一收口）

| 任务域 | 责任主体 | 说明 |
| --- | --- | --- |
| 平台主骨架与目录规则 | ARCH 主控 | 包括目录、脚本入口、共享配置与命名基线。 |
| 环境编排与运行模式 | ARCH 主控 | 包括 `local/staging/demo` 切换与 compose 边界。 |
| 状态机冻结与生命周期口径 | ARCH 主控 | 包括主状态、子状态、状态转换守卫。 |
| 事件模型与 Topic 冻结 | ARCH 主控 | 包括事件名、载荷字段、幂等键与重试策略。 |
| OpenAPI 冻结与错误码字典 | ARCH 主控 | 包括路径、DTO、错误码映射与版本策略。 |

## 可分发任务（在主控边界内并行实现）

| 任务域 | 责任主体 | 约束 |
| --- | --- | --- |
| 模块业务实现（IAM/CAT/TRADE/DLV/BIL/AUD/OPS） | AGENT 执行 | 必须遵循主控冻结的接口、状态机、错误码与事件契约。 |
| 前端页面与流程接线 | AGENT 执行 | 仅消费已冻结 API，不反向定义后端契约。 |
| 测试用例实现 | AGENT 执行 | 用例矩阵与验收口径由主控文档定义。 |

## 升级原则

1. 可分发任务若触发接口/事件/状态机变更，必须先回到主控任务更新基线文档。
2. 主控任务变更必须同步更新：实施进度日志、TODO 汇总、对应冻结文档引用。
3. 未经主控收口，不得将临时实现标记为“已完成”。
