# 发布策略（BOOT-020）

本策略定义 `local`、`staging`、`demo` 三套环境的发布粒度、回滚原则、数据重置和演示保护规则。

## 发布粒度

- `local`：允许按模块快速迭代，重点验证开发闭环与脚本可用性。
- `staging`：按批次发布，必须带任务覆盖与验证记录。
- `demo`：按里程碑发布，仅接受已审批通过的批次集合。

## 回滚原则

1. 先回滚应用，再处理数据修复；禁止先删库后回滚代码。
2. 任何涉及迁移的发布都必须有可执行回滚或前向修复方案。
3. 事件发布失败采用补偿与重试，不通过人工改库跳过状态。

## 数据重置策略

- `local`：允许重置，使用固定脚本执行并保留重置日志。
- `staging`：默认不重置，仅在审批通过后执行受控重置。
- `demo`：禁止随意重置；如需重置必须先导出证据包并保留变更单。

## 演示环境保护

- 演示前冻结依赖版本（OpenAPI、模板、fixtures）。
- 禁止在演示窗口直接合并破坏性变更。
- 演示期仅允许修复级变更，并同步审计记录。
