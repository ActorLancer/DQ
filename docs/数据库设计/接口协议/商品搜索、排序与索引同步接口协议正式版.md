# 商品搜索、排序与索引同步接口协议正式版

## 1. 文档定位

本协议用于统一平台在以下接口上的对外语义：

- 商品搜索
- 卖方搜索
- 搜索排序
- 索引同步查询
- 搜索重建
- 搜索缓存失效

本文件与以下文档保持一致：

- [商品搜索、排序与索引同步设计.md](../../原始PRD/商品搜索、排序与索引同步设计.md)
- [权限点清单.md](../../权限设计/权限点清单.md)
- [数据库表字典正式版.md](../数据库表字典正式版.md)

## 2. 协议原则

- PostgreSQL 是主数据权威源
- OpenSearch 返回候选结果与排序得分
- 最终业务可见性和可售性由 PostgreSQL 校验
- Redis 仅做缓存
- 所有搜索运维写接口必须带 `X-Idempotency-Key`

## 3. 公共请求头

- `X-Request-Id`
- `X-Idempotency-Key`
- `Authorization: Bearer <access_token>`

## 4. V1 接口

### 4.1 搜索与详情

- `GET /api/v1/catalog/search`
- `GET /api/v1/sellers/{orgId}/profile`

### 4.2 搜索运维

- `GET /api/v1/ops/search/sync`
- `POST /api/v1/ops/search/reindex`
- `POST /api/v1/ops/search/aliases/switch`
- `POST /api/v1/ops/search/cache/invalidate`
- `GET /api/v1/ops/search/ranking-profiles`
- `PATCH /api/v1/ops/search/ranking-profiles/{id}`

## 5. V2 接口增量

- `GET /api/v2/catalog/search/hybrid`
- `GET /api/v2/models/search`
- `POST /api/v2/ops/search/synonyms`

## 6. V3 接口增量

- `GET /api/v3/ecosystem/search`

## 7. 请求语义建议

### 7.1 `GET /api/v1/catalog/search`

关键参数：

- `q`
- `entity_scope=all|product|service|seller`
- `industry`
- `tags`
- `delivery_mode`
- `price_min`
- `price_max`
- `sort=composite|latest|price_asc|price_desc|quality|reputation|hotness`
- `page`
- `page_size`

### 7.2 `POST /api/v1/ops/search/reindex`

关键参数：

- `entity_scope`
- `target_backend=opensearch`
- `mode=single|batch|full`
- `force`

## 8. 错误码建议

- `SEARCH_QUERY_INVALID`
- `SEARCH_BACKEND_UNAVAILABLE`
- `SEARCH_RESULT_STALE`
- `SEARCH_REINDEX_FORBIDDEN`
- `SEARCH_ALIAS_SWITCH_FORBIDDEN`
- `SEARCH_CACHE_INVALIDATE_FORBIDDEN`

## 9. 一句话结论

搜索接口必须把“查询语义、排序语义、索引运维语义”统一起来，但最终业务状态仍由 PostgreSQL 校验。
