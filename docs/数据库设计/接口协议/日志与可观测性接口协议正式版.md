# 日志与可观测性接口协议正式版

## 1. 文档范围

本协议定义平台在 `V1 / V2 / V3` 的日志、追踪、告警、事件工单和 SLO 管理接口基线。

适用对象：

- 可观测性总览
- 日志检索
- Trace 联查
- 告警中心
- 事件工单
- SLO 管理

## 2. 总体原则

- 审计接口与运维观测接口分离
- 所有观测接口必须带 `X-Request-Id`
- 高风险接口必须带 `X-Step-Up-Token`
- 原始日志导出与 backend 配置修改必须再审计

## 3. 查询接口

### 3.1 可观测性总览

`GET /api/v1/ops/observability/overview`

返回：

- backend 状态
- 最近告警统计
- 关键服务状态
- SLO 摘要
- 最近 incident

### 3.2 日志检索

`GET /api/v1/ops/logs/query`

参数：

- `service_name`
- `log_level`
- `request_id`
- `trace_id`
- `object_type`
- `object_id`
- `from`
- `to`
- `query`

### 3.3 Trace 联查

`GET /api/v1/ops/traces/{traceId}`

返回：

- `trace_id`
- `traceparent`
- `root_service_name`
- `root_span_name`
- `request_id`
- `ref_type`
- `ref_id`
- `started_at`
- `ended_at`

### 3.4 告警中心

`GET /api/v1/ops/alerts`

参数：

- `alert_status`
- `severity`
- `source_backend_key`
- `alert_type`

### 3.5 事件工单

`GET /api/v1/ops/incidents`

参数：

- `incident_status`
- `severity`
- `owner_role_key`

### 3.6 SLO

`GET /api/v1/ops/slos`

## 4. 管理接口

### 4.1 告警确认

`POST /api/v1/ops/alerts/{alertEventId}/ack`

### 4.2 告警规则更新

`PATCH /api/v1/ops/alerts/rules/{alertRuleId}`

### 4.3 创建事件工单

`POST /api/v1/ops/incidents`

### 4.4 更新事件工单

`PATCH /api/v1/ops/incidents/{incidentTicketId}`

### 4.5 更新 SLO

`PATCH /api/v1/ops/slos/{sloDefinitionId}`

### 4.6 观测后端配置

`PATCH /api/v1/ops/observability/backends/{backendKey}`

### 4.7 日志导出

`POST /api/v1/ops/logs/export`

## 5. 鉴权要求

- `GET /api/v1/ops/observability/overview` -> `ops.observability.read`
- `GET /api/v1/ops/logs/query` -> `ops.log.query`
- `POST /api/v1/ops/logs/export` -> `ops.log.export`
- `GET /api/v1/ops/traces/{traceId}` -> `ops.trace.read`
- `GET /api/v1/ops/alerts` -> `ops.alert.read`
- `POST /api/v1/ops/alerts/{id}/ack` -> `ops.alert.ack`
- `PATCH /api/v1/ops/alerts/rules/{id}` -> `ops.alert.manage`
- `GET /api/v1/ops/incidents` -> `ops.incident.read`
- `POST /api/v1/ops/incidents` / `PATCH /api/v1/ops/incidents/{id}` -> `ops.incident.manage`
- `GET /api/v1/ops/slos` -> `ops.slo.read`
- `PATCH /api/v1/ops/slos/{id}` -> `ops.slo.manage`
- `PATCH /api/v1/ops/observability/backends/{key}` -> `ops.observability.manage`

## 6. 审计要求

以下动作必须进入高优先级审计：

- 原始日志导出
- backend 配置修改
- 告警规则修改
- 工单强制关闭
- SLO 阈值修改

## 7. 闭环说明

日志与可观测性接口只负责运行域联查与运维治理，不替代：

- `audit.*` 证据域
- `trade.* / payment.* / delivery.*` 业务权威域
- `Fabric` 可信证明确认层
