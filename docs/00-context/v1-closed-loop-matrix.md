# V1-Core 闭环矩阵（CTX-021）

## 1. 矩阵说明

- 维度：`8 SKU × 5 标准链路`
- 每个交叉点统一给出：
  - 主触发点
  - 状态推进点（覆盖合同/授权/交付/验收/计费/退款/争议/审计）
  - 证据对象
  - 测试入口
- 标记规则：
  - `主挂点`：该链路的主 SKU
  - `补充挂点`：该链路允许的补充 SKU
  - `非挂点`：不作为该链路 V1 主验收路径，仅保留边界一致性检查

## 7. 使用约束

1. 非挂点交叉项不等于“可忽略”，必须至少具备策略阻断与审计记录。
2. 所有主挂点与补充挂点都必须遵守“主状态在 PostgreSQL、异步副作用走 outbox/Kafka、结果可审计联查”的统一规则。
