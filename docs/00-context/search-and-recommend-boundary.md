# 搜索与推荐边界冻结（CTX-011）

## 1. 数据权威

- `PostgreSQL` 是搜索与推荐放行前的最终业务校验源。
- `OpenSearch` 只承担搜索读模型、召回与排序辅助。
- 推荐缓存（含 `Redis`）只承担短期候选与性能优化，不是业务真值。

## 2. 放行规则

搜索/推荐结果返回前，必须至少完成：

1. 可见性校验
2. 主体资格校验
3. 合同/策略约束校验
4. 风控/冻结校验
5. `PostgreSQL` 最终状态校验

## 3. 禁止事项

- 禁止仅凭 `OpenSearch` 命中结果直接放行交易入口。
- 禁止在推荐模块单独实现一套与平台冲突的放行规则。
- 禁止将搜索索引状态替代商品主状态。
