# 首批五条标准链路冻结（CTX-007）

## 1. 标准链路与主 SKU

| 编号 | 标准链路 | 主 SKU | 可选补充 SKU |
| --- | --- | --- | --- |
| S1 | 工业设备运行指标 API 订阅 | `API_SUB` | `API_PPU` |
| S2 | 工业质量与产线日报文件包交付 | `FILE_STD` | `FILE_SUB` |
| S3 | 供应链协同查询沙箱 | `SBX_STD` | `SHARE_RO` |
| S4 | 零售门店经营分析 API / 报告订阅 | `API_SUB` | `RPT_STD` |
| S5 | 商圈/门店选址查询服务 | `QRY_LITE` | `RPT_STD` |

## 2. 验收路径（最小）

- S1：订阅开通 -> 调用授权生效 -> 周期计费可追踪。
- S2：交付对象生成 -> 下载/回执 -> 验收状态可推进。
- S3：沙箱席位开通 -> 查询执行 -> 结果导出边界受控。
- S4：API 订阅生效 -> 周期交付或报告交付 -> 账单正确。
- S5：模板查询授权 -> 查询执行成功 -> 结果包签收/回执有效。

## 3. 演示数据需求（最小）

- 每条链路至少准备 1 个卖方主体、1 个买方主体、1 组可交易 SKU、1 条可验证订单样本。
- 每条链路必须能联查主对象状态与审计轨迹（`request_id` / 关键对象 ID）。
- 演示样本必须覆盖全部 8 个 V1 SKU 的落点，不允许存在“冻结 SKU 无场景挂点”。
