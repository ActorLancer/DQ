# 生命周期对象冻结（CTX-005）

## 1. 冻结对象清单

`V1-Core` 生命周期主对象固定为以下七类：

1. `Order`
2. `DigitalContract`
3. `Authorization`
4. `Delivery`
5. `Settlement`
6. `Dispute`
7. `BillingEvent`

## 2. 执行约束

- 后续模块命名、状态机和审计追踪必须围绕这七类对象展开。
- 禁止在交易主链路外另建平行生命周期主对象。
- 外围进程可产生外部事实，但不得替代主对象状态推进。

## 3. 最小状态一致性要求

- 每次关键状态推进都应可回溯到：
  - 主对象变更
  - 审计记录
  - outbox 事件
- 链回执、支付回执、交付回执等外部事实必须挂接到上述对象，不得脱离对象体系独立演进。
