# 安全与审计最低基线冻结（CTX-012）

## 1. 最低基线

`V1-Core` 至少必须统一落地以下安全与审计底线：

1. 统一 `request_id` 贯穿接口、事件、审计与运维联查
2. 关键对象 ID 全链路可追踪（订单、合同、授权、交付、结算、争议、账单事件）
3. 高风险动作强制审计留痕
4. 审计导出动作必须记录“谁、何时、为何、导出范围”
5. 高风险操作需要再认证（step-up）
6. SoD（职责分离）动作需双人复核或等效机制
7. 默认最小披露，避免超范围暴露对象和敏感字段

## 2. 高风险动作范围（最小）

- 退款、赔付、人工打款
- 审计包导出与重放
- 一致性修复、死信重处理
- 索引重建、别名切换、全量缓存失效
- 主体/商品/应用冻结解冻

## 3. 审计字段最小集合

- `request_id`
- `actor_id`
- `permission_code`
- `object_type`
- `object_id`
- `result_code`
- `risk_flags`
