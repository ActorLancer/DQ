# V1-Core 范围边界冻结（CTX-003）

## 1. V1-Core 仅做什么

`V1-Core` 只实现最小可信交易闭环，覆盖标准交易主路径：

- 主体与访问基础能力（认证、会话、权限校验）
- 目录与商品基础路径（上架、审核、可见性）
- 交易主链路（下单、合同、授权、交付、验收、结算、争议）
- 支付回调幂等与对账基础
- 审计、证据、一致性回写基础
- 搜索/推荐的 V1 读模型同步与主库回查

## 2. V1-Core 明确不做什么

以下能力不进入 `V1` 正式实现，只允许边界预留：

- `MPC`
- `TEE`
- `FL`
- `ZKP`
- `C2D`（受控计算正式交易能力）
- 生产级公链锚定依赖
- 自然人供给市场与跨境自动交付

## 3. V2/V3 处理规则

1. 可保留扩展点，但不得提前落正式能力。
2. 若当前任务必须预留，必须使用标准注释：
   - `TODO(V2-reserved, TASK-ID): ...`
   - `TODO(V3-reserved, TASK-ID): ...`
3. 任何应在当前批次完成却未完成的事项必须标记：
   - `TODO(V1-gap, TASK-ID): ...`
