# 任务 Ownership 策略冻结（CTX-013）

## 1. 目标

统一后续任务责任边界，避免多个执行者按不同拆分方式并行改造导致模块漂移。

## 2. 所有权原则

1. 前置统筹阶段优先负责：
   - 基础架构边界冻结
   - 仓库骨架与目录收敛
   - `platform-core` 主骨架约束
   - 领域边界冻结文档
2. 业务模块实现必须严格按 `depends_on` 推进，不得跨依赖跳做。
3. 主状态推进权固定在 `platform-core`，外围进程仅承担适配与异步副作用。

## 3. 任务分配规则

- `CTX/BOOT/ENV/CORE` 任务优先解决“边界、骨架、运行基线”，作为业务域前置。
- 业务域任务（`IAM/CAT/TRADE/DLV/BIL/AUD/SEARCHREC`）按依赖链串行推进。
- 每批次只允许“1 个复杂任务”或“2~4 个连续简单任务”，并执行半人工审批门禁。

## 4. 禁止事项

- 未满足前置依赖时抢跑后续域任务。
- 未经审批通过继续进入下一批。
- 未经边界冻结就新增平行模块或平行服务拆分。
