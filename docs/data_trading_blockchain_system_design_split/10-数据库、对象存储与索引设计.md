# 10. 数据库、对象存储与索引设计

## 10.1 关系数据库主表

**数据库主表（建议）**

| **表名**            | **关键字段**                                                      | **说明**              |
|---------------------|-------------------------------------------------------------------|-----------------------|
| org_account         | org_id, org_name, org_type, status, compliance_level              | 组织主数据。          |
| user_account        | user_id, org_id, login_id, role_set, status                       | 用户主数据。          |
| did_binding         | did_id, org_id, user_id, chain_id, cert_sn                        | 链上身份映射。        |
| data_asset          | asset_id, owner_org_id, title, category, sensitivity_level        | 数据资产主表。        |
| asset_version       | asset_version_id, asset_id, version_no, schema_hash, full_hash    | 版本表。              |
| product_catalog     | product_id, asset_version_id, seller_org_id, price, status        | 商品表。              |
| order_main          | order_id, product_id, buyer_org_id, seller_org_id, status, amount | 订单主表。            |
| escrow_ledger       | ledger_id, order_id, lock_amount, deposit_amount, status          | 托管记录。            |
| delivery_record     | delivery_id, order_id, object_id, envelope_id, receipt_hash       | 交付记录。            |
| dispute_case        | case_id, order_id, reason_code, status, decision_code             | 争议表。              |
| rating_record       | rating_id, order_id, rater_id, target_id, score, weight           | 评分表。              |
| reputation_snapshot | snapshot_id, subject_id, score, risk_level, effective_at          | 信誉快照。            |
| training_task       | task_id, initiator_org_id, task_type, status, policy_hash         | 训练任务（Phase 2）。 |
| model_update        | update_id, task_id, round_no, participant_id, update_hash         | 模型更新（Phase 2）。 |
| cross_chain_request | ccr_id, source_chain, target_chain, status, payload_hash          | 跨链请求（Phase 3）。 |
| risk_alert          | alert_id, subject_id, alert_type, score, status                   | 风险告警（Phase 3）。 |
| audit_event         | audit_id, domain, ref_id, actor_id, action, event_time            | 审计主表。            |

## 10.3 搜索索引设计

**建议建立的索引**

| **索引对象** | **用途**                            | **注意事项**                       |
|--------------|-------------------------------------|------------------------------------|
| 商品目录索引 | 支持分类、标签、全文描述检索。      | 敏感字段不直接入索引。             |
| 订单检索索引 | 支持按状态、主体、时间、金额检索。  | 以数据库投影为主。                 |
| 审计日志索引 | 支持按 actor、ref_id、action 检索。 | 写入脱敏后的结构化日志。           |
| 风险图索引   | 支撑图分析和关系查询。              | Phase 3 使用图数据库或图计算引擎。 |
