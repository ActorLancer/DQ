# V1-Core 实施进度日志（入口页）

本文件是 `V1-Core` 实施进度日志的唯一入口页，不承载完整批次正文。

正式规则冻结如下：

- 实施日志按分卷记录，正文保存在 `V1-Core-实施进度日志-P*.md`
- 流程文档、README、TODO、审批模板统一只引用本入口页
- 执行 Agent 必须先阅读本入口页，再进入当前活动分卷读写批次正文
- 若后续需要新增分卷，必须先更新本入口页，再开始写入新分卷

## Agent 读写规则

1. 先阅读本入口页，确认当前活动分卷。
2. 新批次的“计划中 / 待审批 / 返工”正文统一追加到当前活动分卷。
3. 人工审批、TODO 追溯、流程换手统一通过本入口页定位对应分卷。
4. 不允许在流程文档中直接写死某个分卷作为长期固定入口。

## 分卷命名规则

- 新分卷统一命名为 `docs/开发任务/V1-Core-实施进度日志-P{N}.md`
- 新分卷启用前，必须先在本入口页更新：
  - 当前活动分卷
  - 历史分卷列表
  - 如有需要，补充分卷适用范围说明
