# AI Agent 执行提示词

你当前负责实现一个区块链数据交易平台的 `V1-Core` 最小可信交易闭环。
开发必须严格遵守现有冻结文档，不得擅自扩展业务范围，不得引入 `V2/V3` 的正式实现。

## 一、先读这些文件

1. `全集成文档/数据交易平台-全集成基线-V1.md`
2. `开发准备/服务清单与服务边界正式版.md`
3. `开发准备/接口清单与OpenAPI-Schema冻结表.md`
4. `开发准备/事件模型与Topic清单正式版.md`
5. `开发准备/统一错误码字典正式版.md`
6. `开发准备/测试用例矩阵正式版.md`
7. `开发任务/v1-core-开发任务清单.csv`
8. `开发任务/v1-core-开发任务清单.md`
9. `开发任务/Agent-开发与半人工审核流程.md`
10. `开发任务/V1-Core-实施进度日志.md`（入口页；再进入当前活动分卷）
11. `开发任务/V1-Core-TODO与预留清单.md`
12. `开发任务/V1-Core-人工审批记录.md`
13. `开发任务/问题修复任务/README.md`

## 二、执行源规则

- `开发任务/v1-core-开发任务清单.csv` 是唯一执行源。
- `开发任务/v1-core-开发任务清单.md` 只是阅读版，用于人工审阅和上下文理解。
- `开发任务/Agent-开发与半人工审核流程.md` 是唯一执行流程源。
- 若 CSV 与 Markdown 存在任何表述差异，必须以 CSV 为准。
- CSV 行序主要服务阅读与分段展示；若行序与 `depends_on` 冲突，必须以 `depends_on` 作为执行顺序的唯一硬约束。
- 当前任务状态采用 **`NOTIF` 为切线的二态规则**：
  - `NOTIF` 之前的任务统一记为 `yes`
  - `NOTIF` 及之后的任务统一记为 `no`
- `NOTIF` 及之后的任务即便已有占位代码、README、runbook、OpenAPI 草稿或测试 README，也仍视为未完成；这些资产只用于冻结边界与承接后续开发，不构成任务完成。
- 若任务 `technical_reference` 引用了 `开发任务/问题修复任务/A*.md`，必须先阅读该文档的 `## 0. 当前状态`；`## 2. 历史问题起点（归档）` 和 `## 4. 历史问题证据（归档）` 只用于理解问题发现时的历史背景，不得直接视为当前实现状态。
- 若当前阶段无法实现的功能，使用 TODO 注释 + 详细完整的解释注释标注并同步到 [TODO 列表](V1-Core-TODO与预留清单.md) 中，绝对不允许简化实现任何功能。

## 三、你的执行规则

- 只实现当前被分配的任务，不擅自跨任务扩展。
- 实现前先确认前置任务是否已经完成；若未完成，必须暂停并等待下一步指示。
- 每次只允许完成 1 个复杂任务，或 2 到 4 个连续简单任务组成的一个批次。
- 每次必须显式引用当前任务编号和其 `depends_on`。
- 编码前必须先读取 `开发任务/V1-Core-实施进度日志.md`，确认当前活动分卷，再在该分卷中写一条“计划中”记录。
- 每完成一个任务，必须同时完成：
  - 功能实现
  - 自检
  - 与冻结文档逐项对照
  - 与当前任务清单逐项对照
  - 缺口说明
- 更新 `开发任务/V1-Core-实施进度日志.md` 指向的当前活动分卷；若切换到新分卷，必须先更新入口页。
- 更新 `开发任务/V1-Core-TODO与预留清单.md`
- 等待人工在 `开发任务/V1-Core-人工审批记录.md` 中给出审批结果
- 若实现结果没有完整覆盖任务要求，不得标记完成。
- 若发现任务描述与冻结文档冲突，不得自行裁决，必须暂停并上报。
- 当前批次未审批通过前，不得推进下一批任务。
- 不允许把 `V1-gap` 伪装成 `V2/V3-reserved`。

## 四、每个任务完成后必须输出

1. 当前任务编号
2. 前置任务核对结果
3. 已实现内容
4. 涉及文件
5. 验证步骤
6. 验证结果
7. 对应冻结文档覆盖项
8. 对应任务清单覆盖项
9. 未覆盖项
10. 是否可以进入下一任务
11. 新增 TODO / 预留项
12. 当前批次是否已写入实施日志、TODO 清单、人工审批记录

以上内容必须同步写入 `开发任务/V1-Core-实施进度日志.md` 指向的当前活动分卷。

若当前阶段未实现但必须预留：

- 代码中必须写 `TODO(V1-gap, TASK-ID)`、`TODO(V2-reserved, TASK-ID)` 或 `TODO(V3-reserved, TASK-ID)`
- 同时必须登记到 `开发任务/V1-Core-TODO与预留清单.md`

## 五、严格暂停条件

以下任一情况出现时，必须暂停：

- 前置任务未完成
- 依赖服务未就绪
- 数据库迁移未通过
- 接口契约与实现不一致
- 事件契约与实现不一致
- 权限校验未落地
- 当前任务无法完整覆盖其开发范围
- 当前任务涉及的代码、配置、文档同步未完成
- 当前批次未写实施日志
- 当前批次未更新 TODO 汇总
- 当前批次尚未获得人工审批通过

## 六、完成定义

只有同时满足以下条件，任务才算完成：

- 功能已实现
- 冻结文档覆盖完整
- 当前任务清单覆盖完整
- 自测通过
- 相关接口/事件/权限/数据库变更已同步
- 没有遗留必须立即处理的缺口
- 前置和后续影响已说明
- 实施日志已更新
- TODO 汇总已更新
- 人工审批记录已更新且审批结论为“通过”
