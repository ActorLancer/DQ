# 问题修复任务文档使用规则

本目录中的 `A01 ~ A15` 文档属于“问题治理与剩余边界说明文档”，不是当前实现状态的唯一事实源。

## 阅读顺序

1. 先读 `docs/开发任务/v1-core-开发任务清单.csv`
2. 再读对应任务的 README / runbook / OpenAPI / test-case / TODO
3. 若任务 `technical_reference` 引用本目录文档，再阅读对应 `Axx` 文档的：
   - `## 0. 当前状态`
   - `## 1. 任务定位`
   - `## 5. 任务目标`
   - `## 6. 强约束`
   - `## 7. 建议修复方案`
   - `## 8. 实施范围`
   - `## 9. 验收标准`

## 时态约束

- `## 2. 历史问题起点（归档）` 与 `## 4. 历史问题证据（归档）` 记录的是问题发现时的历史状态。
- 这些归档段落用于解释“为什么会出现该问题”，不直接代表当前实现状态。
- 当前实现状态、当前运行规则和后续执行义务，必须优先以：
  - `CSV`
  - `Markdown 阅读版任务清单`
  - `TODO`
  - `README / runbook / scripts`
  为准。

## 使用原则

- 如果某份 `Axx` 文档继续被任务清单引用，它必须同时包含：
  - 当前状态
  - 历史问题起点
  - 当前已收缩部分
  - 剩余未完成项或后续承接关系
- 不允许继续把“历史问题起点”写成当前现状。
- 后续新增 `A16+` 文档时，也必须遵守同一结构。
