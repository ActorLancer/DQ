# V1-Core 人工审批记录

本文件用于记录人工对每个实现批次的审批结果。

## 审批规则

- 每个批次必须有一条审批记录
- 未审批通过，不得进入下一批
- 若结论为“需补充后复审”或“驳回并返工”，执行 Agent 必须先修当前批
- 审批记录应引用对应的实施日志批次编号

---

## 审批模板

### BATCH-XXX

- 对应任务编号：
- 审核依据：
- 审核结论：通过 / 需补充后复审 / 驳回并返工
- 当前批次状态：
- 已完成确认：
- 偏离目标点：
- 未实现项：
- TODO 是否充分：
- 要求补充内容：
- 是否允许进入下一批：yes / no
- 审批人备注：

## 最小审批结论规则

- `通过`
  - 当前批次可关闭，允许进入下一批
- `需补充后复审`
  - 当前批次不关闭，只允许修补当前问题
- `驳回并返工`
  - 当前批次必须重做关键内容，不得保留“已完成”口径
