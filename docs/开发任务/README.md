# 开发任务目录

当前目录用于分发给其他 AI Agent 或人工开发者执行的任务清单。

阅读顺序：

1. [v1-core-开发任务清单.csv](./v1-core-开发任务清单.csv)
2. [v1-core-开发任务清单.md](./v1-core-开发任务清单.md)
3. [Agent-开发与半人工审核流程.md](./Agent-开发与半人工审核流程.md)
4. [AI-Agent-执行提示词.md](./AI-Agent-执行提示词.md)
5. [V1-Core-实施进度日志.md](./V1-Core-实施进度日志.md)（入口页）
6. [V1-Core-TODO与预留清单.md](./V1-Core-TODO与预留清单.md)
7. [V1-Core-人工审批记录.md](./V1-Core-人工审批记录.md)
8. [问题修复任务/README.md](./问题修复任务/README.md)

执行原则：

- 当前只做 `V1-Core` 最小可信交易闭环。
- 不允许擅自扩展到 `V2/V3` 的正式实现。
- **CSV 是唯一执行源，Markdown 只是阅读版。** 若两者存在表述差异，以 CSV 为准。
- 当前任务状态采用 **`NOTIF` 为切线的二态规则**：
  - `NOTIF` 之前的任务统一记为 `yes`
  - `NOTIF` 及之后的任务统一记为 `no`
- `NOTIF` 及之后即便已有占位代码、文档、README、runbook 或 OpenAPI 草稿，也不构成任务完成；后续需在正式实现批次中重新开发、验证和验收。
- 实施流程以 [Agent-开发与半人工审核流程.md](./Agent-开发与半人工审核流程.md) 为准。
- 实施日志入口固定为 [V1-Core-实施进度日志.md](./V1-Core-实施进度日志.md)，详细批次正文按 `V1-Core-实施进度日志-P*.md` 分卷保存。
- `问题修复任务/A*.md` 属于问题治理与剩余边界说明文档；若任务 `technical_reference` 引用了这些文档，必须先看其 `## 0. 当前状态`，再看归档的历史问题起点，不能把历史问题起点误读成当前现状。
- 当前仓库已存在部分骨架、脚本与部署文件，执行任务时优先复用，不允许无意义重建。
- 所有任务完成后必须验证功能是否闭环、实现是否完整覆盖任务要求。
- 每完成一个复杂任务或一批简单任务，都必须更新实施日志、TODO 汇总和人工审批记录。
- 每个批次开始前也必须先写“计划中”日志，再开始编码。
- 若发现前置任务未完成或冻结文档存在硬冲突，必须暂停并上报。
