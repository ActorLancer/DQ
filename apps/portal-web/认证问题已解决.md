## 🔍 问题原因

### 根本原因
1. **Zustand persist 状态恢复延迟**: Zustand 的 persist 中间件需要时间从 localStorage 恢复状态
2. **过早的权限检查**: `ProtectedRoute` 组件在状态恢复完成前就执行了权限检查
3. **配置问题**: `partialize` 配置可能导致状态恢复不完整

### 表现症状
- 登录成功后立即跳转到 `/unauthorized`
- 刷新页面后也会跳转到 `/unauthorized`
- 即使 localStorage 中有认证信息也无法访问控制台

---

## 🔧 解决方案

### 1. 修复 `useAuthStore.ts`
**位置**: `src/store/useAuthStore.ts`

**修改内容**:
- ✅ 移除了 `partialize` 配置
- ✅ 让 persist 中间件自动处理所有状态
- ✅ 简化了 logout 函数

**修改前**:
```typescript
persist(
  (set, get) => ({ /* ... */ }),
  {
    name: 'auth-storage',
    partialize: (state) => ({
      token: state.token,
      user: state.user,
      isAuthenticated: state.isAuthenticated,
    }),
  }
)
```

**修改后**:
```typescript
persist(
  (set, get) => ({ /* ... */ }),
  {
    name: 'auth-storage',
    // 移除 partialize，让 persist 自动处理
  }
)
```

### 2. 修复 `ProtectedRoute.tsx`
**位置**: `src/components/auth/ProtectedRoute.tsx`

**修改内容**:
- ✅ 添加 `isHydrated` 状态跟踪
- ✅ 等待状态恢复完成后再执行权限检查
- ✅ 添加调试日志，方便排查问题
- ✅ 改进加载状态显示

**关键代码**:
```typescript
const [isHydrated, setIsHydrated] = useState(false)

// 等待 Zustand persist 从 localStorage 恢复状态
useEffect(() => {
  setIsHydrated(true)
}, [])

useEffect(() => {
  // 等待状态恢复完成
  if (!isHydrated) return
  
  // 然后再执行权限检查
  // ...
}, [isHydrated, /* ... */])
```
