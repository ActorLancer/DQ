## 📝 核心功能说明

### 1. 认证系统

#### 登录流程
```
访问登录页 → 选择角色 → 输入凭证 → Mock 验证 → 保存状态 → 跳转控制台
```

#### 状态持久化
- 使用 Zustand persist 中间件
- 自动保存到 LocalStorage
- 刷新页面保持登录状态
- 关闭浏览器后重新打开仍保持登录

#### 路由保护
- 未登录访问控制台 → 跳转登录页
- 登录后自动返回原页面（returnUrl）
- 角色不匹配 → 跳转 /unauthorized
- 权限不足 → 跳转 /unauthorized

### 2. 权限控制

#### 角色系统
- **buyer**: 买家，访问买家控制台
- **seller**: 供应商，访问供应商后台
- **admin**: 管理员，访问平台运营后台

#### 权限检查
- `hasRole(role)`: 检查用户是否拥有指定角色
- `hasPermission(permission)`: 检查用户是否拥有指定权限
- 支持细粒度权限控制（如 `buyer:orders:delete`）

### 3. 链证明展示

所有涉及链上操作的页面都展示：
- **Request ID**: 请求唯一标识（等宽字体）
- **TX Hash**: 交易哈希（等宽字体）
- **Chain Status**: 链状态（已确认/未提交/失败）
- **Projection Status**: 投影状态（已同步/未同步）

### 4. 会话管理

#### SessionIdentityBar
- 显示当前主体、角色、租户、作用域
- 显示登录用户
- 会话倒计时（30 分钟）
- 少于 5 分钟时红色闪烁警告
- 支持主体和角色切换入口

### 5. API 密钥管理

#### 安全特性
- 创建时一次性显示完整 Key
- 列表中脱敏显示（`sk_***************abc`）
- 支持轮换、禁用、删除操作
- IP 白名单配置
- 过期时间设置

### 6. 一致性检查（⭐ 核心功能）

#### 三列状态指示器
- **数据库**: 绿色=有记录，红色=无记录
- **区块链**: 绿色=已确认，红色=未提交/失败
- **投影**: 绿色=已同步，红色=未同步

#### 不一致类型
- **PROJECTION_MISSING**: 投影缺失
- **CHAIN_NOT_CONFIRMED**: 链未确认
- **CHAIN_SUBMISSION_FAILED**: 链提交失败
- **DATA_MISMATCH**: 数据不一致

#### 修复操作
- 重新检查
- 尝试修复（重新提交链、重建投影）
