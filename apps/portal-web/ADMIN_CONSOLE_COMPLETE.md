### 4. 系统一致性检查 - ⭐ 核心功能

#### 统计卡片（4 个）
```
✅ 总检查记录: 5 条
✅ 一致记录: 2 条
✅ 不一致记录: 3 条
✅ 一致性率: 40.0%
```

#### 筛选功能
```
✅ 搜索: Request ID / TX Hash / Business ID
✅ 筛选: 全部记录 / 仅一致 / 仅不一致
✅ 运行检查按钮: 带加载动画（旋转图标）
```

#### 检查结果列表（左侧 2/3 宽度）

每个检查记录卡片包含：

1. **头部**:
   - ✅ 业务类型标签（商品/访问申请/订单/订阅/交付）
   - ✅ 一致性状态标签（一致=绿色，不一致=红色）
   - ✅ Request ID（等宽字体）

2. **三列状态指示器**（核心设计）:
   ```
   ┌─────────────┬─────────────┬─────────────┐
   │  数据库     │  区块链     │   投影      │
   │  Database   │  Chain      │  Projection │
   │  ✓ 有记录   │  ✓ 已确认   │  ✓ 已投影   │
   │  (绿色)     │  (绿色)     │  (绿色)     │
   └─────────────┴─────────────┴─────────────┘
   ```
   - ✅ 每列独立卡片
   - ✅ 图标（数据库/链接/盾牌）
   - ✅ 状态文字
   - ✅ 颜色区分（绿色=正常，红色=异常）

3. **不一致类型**（仅不一致记录）:
   - ✅ 红色警告卡片
   - ✅ 不一致类型标题
   - ✅ 详细描述

4. **底部信息**:
   - ✅ 最后检查时间
   - ✅ 修复按钮（仅不一致记录）

#### 检查详情（右侧 1/3 宽度，Sticky）

- ✅ **业务信息**:
  - 业务类型标签
  - Business ID（等宽字体）
  - Request ID（等宽字体）

- ✅ **链上信息**（如有 TX Hash）:
  - TX Hash（等宽字体）
  - 链状态标签

- ✅ **投影状态**:
  - 投影状态标签

- ✅ **一致性状态**（核心卡片）:
  - 一致: 绿色背景 + 对勾图标
  - 不一致: 红色背景 + 警告图标
  - 三项检查结果:
    ```
    数据库记录: ✓ / ✗
    链上记录:   ✓ / ✗
    投影记录:   ✓ / ✗
    ```

- ✅ **不一致详情**（仅不一致记录）:
  - 红色警告卡片
  - 不一致类型标题
  - 详细描述

- ✅ **时间信息**:
  - 创建时间
  - 最后检查时间

- ✅ **操作按钮**:
  - 重新检查（主按钮）
  - 尝试修复（仅不一致记录）

#### 不一致类型定义

| 类型 | 标签 | 描述 |
|------|------|------|
| PROJECTION_MISSING | 投影缺失 | 链上记录已确认，但投影数据库中缺少对应记录 |
| CHAIN_NOT_CONFIRMED | 链未确认 | 数据库有记录，但链上交易尚未确认 |
| CHAIN_SUBMISSION_FAILED | 链提交失败 | 向区块链提交交易失败 |
| DATA_MISMATCH | 数据不一致 | 数据库、链上、投影三者数据内容不一致 |
