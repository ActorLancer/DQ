# 数据交易平台后端鉴权中间件规则说明

## 1. 文档说明

本说明定义后端统一鉴权中间件的职责、处理顺序和返回规范。  
目标是让：

- 网关
- BFF
- 业务服务
- 异步任务消费者

都使用同一套鉴权判断逻辑。

## 2. 鉴权链路总原则

每个请求都必须按以下顺序判断：

1. 身份解析
2. 主体状态校验
3. 基础权限校验
4. 作用域校验
5. 业务对象状态校验
6. 合同/策略校验
7. 环境/网络/配额校验
8. 合规/风控阻断校验
9. 高风险附加校验
10. 审计记录

## 3. 中间件分层

建议分 6 层：

### 3.1 Identity Middleware

职责：

- 解析用户令牌、应用令牌、连接器证书、服务签名
- 提取：
  - `actor_type`
  - `actor_id`
  - `tenant_id`
  - `role_set`
  - `app_id`
  - `cert_sn`

失败返回：

- `AUTH_INVALID_TOKEN`
- `AUTH_CERT_REVOKED`
- `AUTH_SIGNATURE_INVALID`

### 3.2 Subject Status Middleware

职责：

- 校验 Tenant/Organization 状态
- 校验 User / Application / Connector / Environment 是否有效

失败返回：

- `AUTH_TENANT_DISABLED`
- `AUTH_USER_SUSPENDED`
- `AUTH_APP_REVOKED`
- `AUTH_CONNECTOR_DISABLED`

### 3.3 Permission Middleware

职责：

- 根据接口绑定的权限点做基础判定
- 支持单权限或多权限任一满足

失败返回：

- `PERM_DENIED`

### 3.4 Scope Middleware

职责：

- 校验作用域是否覆盖目标对象

示例：

- 订单接口要求 `tenant + order`
- 商品接口要求 `tenant + product`
- 应用接口要求 `tenant + application`
- 监管接口要求 `regulator_scope + object`

失败返回：

- `PERM_SCOPE_MISMATCH`

### 3.5 Policy Middleware

职责：

- 校验合同、Usage Policy、时间窗、地域、导出限制、输出边界

失败返回：

- `POLICY_NOT_EFFECTIVE`
- `POLICY_REGION_BLOCKED`
- `POLICY_OUTPUT_FORBIDDEN`
- `POLICY_USAGE_FORBIDDEN`

### 3.6 Risk Guard Middleware

职责：

- 校验风控阻断、冻结状态、黑名单、监管限制

失败返回：

- `RISK_SUBJECT_FROZEN`
- `RISK_PRODUCT_BLOCKED`
- `RISK_REQUEST_BLOCKED`
- `RISK_EXPORT_BLOCKED`

## 4. 高风险动作附加校验

以下动作进入 `HighRiskGuard`：

- 合规阻断
- 风险冻结/解冻
- 退款/赔付
- 算法白名单放行
- 结果导出
- 分润执行/冻结
- 凭证吊销
- 跨链终止/补偿
- 监管材料导出
- 互认关系变更
- 搜索别名切换
- 搜索全量重建

附加校验项：

- 二次认证状态
- 最近一次 MFA 时间
- 双人复核标记
- 审批票据/治理票据

失败返回：

- `AUTH_MFA_REQUIRED`
- `PERM_SECOND_REVIEW_REQUIRED`
- `GOV_TICKET_REQUIRED`

## 4.1 搜索请求专项规则

- 搜索引擎结果只用于召回与排序，不直接作为最终业务可见性事实
- 搜索接口返回前应再次校验 PostgreSQL 中的商品状态、卖方状态和风险状态
- 搜索运维接口属于平台运维高风险接口，必须强审计
- OpenSearch 或 Redis 异常时允许切换 PostgreSQL fallback 搜索，但不得跳过权限和可见性校验

## 5. 异步任务鉴权

### 5.1 任务生产

任务入队时必须携带：

- `request_id`
- `actor_id`
- `actor_type`
- `tenant_id`
- `permission_code`
- `scope_snapshot`

### 5.2 任务消费

消费者不得默认信任队列消息，仍需校验：

- 服务身份是否合法
- 作用域快照是否仍有效
- 对象状态是否允许继续执行

适用场景：

- 公链锚定
- 分润执行
- 跨链补偿
- 风控冻结同步
- 联邦聚合

## 6. 机器身份鉴权规则

### 6.1 Application

- 校验应用凭证
- 校验应用归属 Tenant
- 校验绑定的 SKU / API / 模型范围
- 校验 IP / 来源 / 配额

### 6.2 Connector

- 校验证书与签名
- 校验连接器版本与状态
- 校验绑定环境与资源范围

### 6.3 Cross-Chain Gateway / Witness

- 校验网关或见证服务独立身份
- 校验签名来源与轮换状态
- 不得使用平台管理员口令替代

## 7. 审计要求

所有鉴权结果都建议写审计，但至少以下必须记录：

- 鉴权成功的高风险动作
- 鉴权失败的高风险动作
- 监管查询
- 跨链治理动作
- 凭证签发/吊销
- 分润执行/冻结

建议审计字段：

- `request_id`
- `actor_id`
- `permission_code`
- `scope_snapshot`
- `object_type`
- `object_id`
- `result_code`
- `risk_flags`

## 8. 错误码返回规范

### 8.1 HTTP 建议

- `401`：身份无效
- `403`：权限不足或策略阻断
- `409`：状态冲突
- `423`：对象被冻结

### 8.2 业务错误码分组

- `AUTH_*`
- `PERM_*`
- `POLICY_*`
- `RISK_*`
- `GOV_*`

## 9. 接口注册建议

建议每个接口在代码中通过元数据声明：

```text
route
permission_codes
scope_type
high_risk
audit_required
policy_required
risk_guard_required
```

示例：

```text
POST /api/v2/profitshare/rewards/{id}/execute
permission_codes = ["profitshare.reward.execute"]
scope_type = "reward"
high_risk = true
audit_required = true
policy_required = false
risk_guard_required = true
```

## 10. 一句话结论

后端鉴权中间件的核心不是“检查有没有某个角色”，而是把身份、权限、作用域、策略、风控和审计连成统一链路。只有这样，前端菜单、按钮、接口和异步任务的权限语义才能一致。

## 11. 存储与信任边界扩展规则

### 11.1 新增校验点

所有涉及数据本体、模型文件、结果导出和跨平台托管的请求，除现有鉴权链外，还必须校验：

- 当前 `storage_mode`
- 当前 `payload_location_type`
- 平台是否允许触达明文
- 是否存在有效连接器/API 绑定
- 是否存在有效信任证据
- 当前导出是否超出 `result_export_policy`

### 11.2 建议错误码

- `TRUST_STORAGE_MODE_MISMATCH`
- `TRUST_PLATFORM_PLAINTEXT_FORBIDDEN`
- `TRUST_CONNECTOR_NOT_READY`
- `TRUST_EVIDENCE_MISSING`
- `TRUST_EXPORT_BOUNDARY_BLOCKED`

## 12. 支付、资金流与轻结算扩展规则

### 12.1 新增校验点

- `payment_intent` 必须校验订单状态、费用快照、渠道能力和幂等键。
- `payment_intent` 必须校验付款司法辖区、收款司法辖区、起步司法辖区和命中的走廊策略。
- `webhook` 必须校验来源、签名、事件唯一性和重复回调。
- `payout` 必须校验结算状态、风控结果、二次确认和审计要求。
- `mock payment` 只允许在非 production 环境执行。

### 12.2 建议错误码

- `PAYMENT_PROVIDER_NOT_AVAILABLE`
- `PAYMENT_JURISDICTION_UNSUPPORTED`
- `PAYMENT_CORRIDOR_BLOCKED`
- `PAYMENT_PAYOUT_PREFERENCE_INVALID`
- `PAYMENT_SIGNATURE_INVALID`
- `PAYMENT_DUPLICATE_WEBHOOK`
- `PAYMENT_STATUS_MISMATCH`
- `PAYMENT_RECONCILIATION_DIFF_OPEN`
- `PAYMENT_MOCK_FORBIDDEN_IN_PRODUCTION`

## 13. 身份认证、注册登录与会话管理扩展规则

### 13.1 认证链路

后端鉴权必须额外支持：

- 本地登录凭证校验
- 企业 OIDC / SAML 外部身份校验
- `MFA` / `step-up` challenge 校验
- `user_session` / `refresh_token_family` / `trusted_device` 状态校验
- Fabric 身份、证书状态与平台主体绑定关系校验

### 13.2 高风险动作放行条件

以下动作除权限外还必须校验最近一次强认证时间：

- 退款、赔付、人工打款
- 走廊策略、SSO 配置、角色边界修改
- Fabric 身份签发、证书吊销
- 高敏下载、训练放行、跨域导出

### 13.3 建议错误码

- `AUTH_SESSION_REVOKED`
- `AUTH_DEVICE_REVOKED`
- `AUTH_STEP_UP_REQUIRED`
- `AUTH_STEP_UP_EXPIRED`
- `AUTH_SSO_CONNECTION_INVALID`
- `AUTH_FABRIC_IDENTITY_NOT_BOUND`
- `AUTH_CERTIFICATE_REVOKED`
- `AUTH_CERTIFICATE_EXPIRED`

## 14. 审计、证据链与回放扩展规则

### 14.1 新增校验点

- `audit.trace` 只能返回摘要视图，除非显式命中原文读取权限
- `audit.raw.read`、`audit.unmasked_view.read`、`audit.package.export`、`audit.replay.execute`、`audit.legal_hold.manage` 必须校验 step-up
- 回放接口默认只允许 `dry_run = true`
- `break-glass` 必须校验理由、审批状态和事后复核标记
- 监管导出必须校验脱敏级别与导出范围

### 14.2 审计访问再审计

所有以下动作都必须二次写入审计链：

- 查看原始证据
- 查看未脱敏视图
- 导出证据包
- 发起回放
- 创建/解除 `legal hold`
- 执行 `break-glass`

### 14.3 建议错误码

- `AUDIT_RAW_VIEW_FORBIDDEN`
- `AUDIT_UNMASKED_VIEW_REQUIRES_STEP_UP`
- `AUDIT_REPLAY_DRY_RUN_ONLY`
- `AUDIT_LEGAL_HOLD_ACTIVE`
- `AUDIT_BREAKGLASS_REASON_REQUIRED`
- `AUDIT_EXPORT_SCOPE_FORBIDDEN`

## 15. 双层权威模型与一致性扩展规则

### 15.1 基本规则

- 后端不得把 Kafka 消息、Fabric 提交结果或外部回执中的任一者单独当作全局唯一主状态。
- 业务接口默认读取数据库业务状态。
- 对外证明接口必须同时展示 `proof_commit_state`。
- 外部事实型对象必须校验 `external_fact_status` 与来源回执。

### 15.2 高风险动作

以下动作除权限外还必须校验 step-up 与审计：

- `ops.consistency.reconcile`
- `ops.dead_letter.reprocess`
- 任何会改写 `reconcile_status`
- 任何会触发补偿重放或人工覆盖确认的动作

### 15.3 建议错误码

- `CONSISTENCY_MISMATCH_DETECTED`
- `CONSISTENCY_REPLAY_DRY_RUN_ONLY`
- `CONSISTENCY_EXTERNAL_FACT_NOT_CONFIRMED`
- `CONSISTENCY_PROOF_NOT_CONFIRMED`
- `OUTBOX_EVENT_LOCKED`
- `DEAD_LETTER_REPROCESS_FORBIDDEN`
