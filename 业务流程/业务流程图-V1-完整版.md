## 2. V1 流程总览图

```text
[组织/用户准入]
    -> [租户、用户、应用、DID/证书建立]
    -> [供方创建数据资产/商品/SKU/模板/元数据]
    -> [平台审核与上架]
    -> [买方搜索、筛选、查看详情、发起下单]
    -> [价格快照 + 买方资金锁定 + 保证金锁定]
    -> [卖方确认交付准备]
    -> [链下密文交付 + 令牌签发 + 回执采集]
    -> [买方验真/验收]
    -> [通过]
         -> [结算/释放保证金/更新信誉]
         -> [审计归档/报表/关闭]
    -> [拒收或异常]
         -> [争议受理]
         -> [自动裁决或人工复核]
         -> [退款/赔付/扣罚/恢复]
         -> [审计归档/关闭]

所有主线同时伴随：
- [合规审查]
- [风控扫描]
- [链上链下一致性]
- [审计证据链]
- [商品搜索与索引同步]
- [日志/监控/告警]
- [开发者调试与联查]
```

## 3. 参与方泳道

```text
泳道 A: 平台门户/工作台
泳道 B: 身份与权限服务
泳道 C: 商品目录/模板/搜索服务
泳道 D: 订单与账单服务
泳道 E: 交付服务/对象存储/下载网关
泳道 F: 合规风控服务
泳道 G: 联盟链合约/链适配器
泳道 H: 审计与证据服务
泳道 I: 运营/客服/管理员
泳道 J: 开发者通道/调试控制台
```

## 4. 主业务流程

### 4.1 主体准入与基础身份建立

```text
[组织发起注册]
    -> 平台门户采集组织信息、实名资料、联系人
    -> 身份服务创建 Tenant/Party 草稿
    -> 合规服务校验主体资质、行业标签、风险标记
    -> 审核通过?
       -> 否: 退回补资料/拒绝准入/记录审计事件
       -> 是:
          -> 创建 Tenant
          -> 创建管理员 User
          -> 绑定 DID/证书/应用凭证
          -> 初始化角色模板、作用域、默认策略
          -> 输出 org_id / user_id / did_id / app_id
          -> 审计服务固化主体准入证据
```

### 4.2 商品创建、模板绑定与上架流程

```text
[供方进入商品工作台]
    -> 创建 Data Asset
    -> 创建原始接入批次 RawIngestBatch
    -> 登记原始对象清单 RawObjectManifest:
       -> 原始文件/对象 URI
       -> 来源与采集时间
       -> 权利链与责任主体
       -> 原始 hash / 大小 / 格式声明
    -> 触发格式识别与分类:
       -> 结构化 / 半结构化
       -> 文档 / 图片 / 音视频
       -> 数据库转储 / 模型 / Notebook
    -> 进入类内标准化:
       -> 提取 schema / 字段路径 / 章节索引 / 标签摘要 / 预览图
       -> 保留原始格式，不强制统一编码
    -> 进入加工处理区:
       -> 清洗 / 去重 / 缺失修复
       -> 脱敏 / 标注 / 转码 / OCR / ASR / 摘要抽取
       -> 记录 AssetProcessingJob / Input / 证据 hash
    -> 选择商品类型: 文件快照 / 版本订阅 / 只读共享 / API服务 / 模板查询lite / 查询沙箱 / 结果产品
    -> 选择标准 SKU 真值:
       -> FILE_STD / FILE_SUB / SHARE_RO / API_SUB / API_PPU / QRY_LITE / SBX_STD / RPT_STD
    -> 如需售卖套餐:
       -> 生成套餐编码 sku_code
       -> 绑定标准 SKU 类型 sku_type
       -> 套餐编码不得替代标准 SKU 真值
    -> 绑定合同模板、验收模板、退款模板、计费模板
       -> SHARE_RO 必须绑定独立共享模板族
       -> QRY_LITE 与 SBX_STD 必须分别绑定模板
    -> 建立元信息档案:
       -> 业务描述
       -> 结构描述
       -> 质量描述
       -> 安全与合规描述
       -> 交付与版本描述
       -> 验收/授权/责任描述
    -> 维护字段结构说明与主键/时间字段摘要
    -> 生成质量报告摘要与样本展示策略
    -> 维护加工责任链: 原始提供者 / 加工者 / 平台 / 第三方评估方
    -> 生成数据契约草稿: 交付义务 / 验收口径 / 输出边界 / 争议口径
    -> 填写上架元数据: 标题/分类/标签/样本摘要/full_hash/价格/地域/用途/敏感等级
    -> 生成样例/预览工件:
       -> schema preview / sample
       -> text preview / image preview / query preview
       -> preview policy / masking status
    -> 上传样本或密文对象占位 / 外部交付证明
    -> 保存草稿
    -> 提交审核
    -> 合规服务执行:
       -> 数据分类分级检查
       -> 用途限制检查
       -> 地域限制检查
       -> 敏感字段与高风险标签检查
    -> 风控服务执行:
       -> 主体信誉检查
       -> 黑名单/异常历史检查
       -> 商品重复/篡改风险检查
    -> 审核通过?
       -> 否: 退回修改/阻断上架/记录原因
       -> 是:
          -> CatalogService 状态切为 listed
          -> 链适配器调用 ProductRegistry.createProduct
          -> 链上记录 product_id / asset_version_hash / seller_did / price_snapshot / valid_until
          -> 事件总线发出 ProductCreated / ProductListed
          -> 搜索索引刷新
          -> 审计服务固化上架证据包
```

### 4.2A 数据原样处理与产品化加工主流程

```text
[原始数据进入平台治理链]
    -> 原始接入区:
       -> 记录批次、对象、来源、权利链、hash
    -> 分类识别区:
       -> 自动识别对象族与格式
       -> 形成 FormatDetectionResult
    -> 类内标准化区:
       -> 表格类抽 schema/字段
       -> 文档类抽文本/章节
       -> 媒体类抽时长/尺寸/标签
       -> 模型类抽模型卡/依赖
    -> 加工处理区:
       -> 选择平台加工 / 卖方加工 / 第三方加工
       -> 写入 AssetProcessingJob / Input / evidence
    -> 产品包装区:
       -> 生成 asset_version / metadata_profile / quality_report / data_contract
       -> 生成 preview_artifact / sample / manifest
    -> 交付准备区:
       -> 平台托管: 校验密文可用性与票据路径
       -> 卖方自持: 校验连接器/API/交付工单
       -> 受控交付: 校验输出边界与撤权方式
```

### 4.2B 数据商品分层存储闭环

```text
[对象进入存储治理]
    -> 控制面 PostgreSQL:
       -> 记录 StorageNamespace / StoragePolicy / StorageBinding / Hash / 生命周期
    -> 数据面对象存储:
       -> 原始对象写 raw
       -> 加工中间对象写 staging/temp
       -> 可售正式对象写 product
       -> 样例与预览写 preview
       -> 临时交付对象写 delivery
       -> 证据原文写 evidence
       -> 归档对象写 archive
    -> 查询/执行面:
       -> 仅读取 curated / product
       -> 不直接暴露 raw
    -> 信任面 Fabric:
       -> 锚定对象摘要 / 版本摘要 / 订单摘要 / 验收摘要
```

### 4.3 买方搜索、选购与下单流程

```text
[买方搜索目录]
    -> Search API 接收 keyword / filters / entity_scope(product|service|seller|all) / sort_by
    -> Redis 查询热缓存
       -> 命中: 返回缓存结果
       -> 未命中:
          -> 调用 OpenSearch 读取商品/服务/卖方索引
          -> 返回候选 ids + facets + score
          -> Search API 回 PostgreSQL 校验:
             -> 商品状态和卖方状态
             -> 风险冻结状态
             -> 价格与币种快照
             -> 可见性与最小披露
          -> 过滤失效结果
          -> 写回 Redis 热缓存
    -> 搜索服务基于全文字段、精确筛选字段、综合排序字段返回结果
    -> 买方查看详情页:
       -> 商品元数据
       -> 字段结构摘要
       -> 质量报告摘要
       -> 加工责任链摘要
       -> 数据契约摘要
       -> 样本摘要
       -> 使用权利边界
       -> 计费规则
       -> SLA / 退款规则
       -> 合规限制
    -> 买方发起下单
    -> 订单服务执行:
       -> 校验商品状态必须为 listed
       -> 快照 SKU/模板/价格/权利/有效期
       -> 生成 order_id
       -> 计算 buyer_deposit
       -> 生成账单事件草稿
    -> 资金锁定流程:
       -> 买方锁定货款
       -> 买方锁定保证金
       -> 卖方保证金是否已满足?
          -> 否: 要求补足卖方保证金
          -> 是: 继续
    -> 链适配器调用 OrderEscrow.createOrder + lockSellerDeposit
    -> 链上 OrderCreated / BuyerLocked 事件返回
    -> 事件同步器投影 order_main = buyer_locked
    -> 通知卖方待交付
    -> 审计服务记录订单创建、价格快照、资金锁定、request_id、tx_hash
```

### 4.3.1 卖方主页查看流程

```text
[买方在搜索结果中点击卖方主体]
    -> Search API 查询 seller_search_document
    -> PostgreSQL 校验组织状态、公开展示策略和风险状态
    -> 返回:
       -> 卖方主体信息
       -> 行业标签
       -> 信誉等级 / 风险等级
       -> 在售商品数量
       -> 近期成交与争议摘要
    -> 买方可从卖方主页继续筛选其数据商品与服务商品
```

### 4.4 交付、验真与验收主流程

#### 4.4.1 文件类交付

```text
[卖方收到待交付通知]
    -> 交付服务创建 delivery_id
    -> 卖方上传 AES-GCM 密文对象至对象存储
    -> 生成 key_envelope，使用买方公钥封装密钥
    -> 生成 delivery_ticket，设置 expire_at 和 download_limit
    -> 交付网关生成 delivery_commit_hash
    -> 链适配器调用 OrderEscrow.commitDelivery
    -> 事件同步器投影 order_main = delivered
    -> 买方获取下载令牌
    -> 下载网关校验 token、buyer_did、次数、时效
    -> 返回密文对象 + key_envelope
    -> 买方本地解密并重算 Hash
    -> Hash 一致?
       -> 是: 按数据契约与质量报告执行验收校验后提交 accept
       -> 否: 提交 reject 或 open_case
```

#### 4.4.1A 版本订阅交付

```text
[买方购买 revision / 版本订阅 SKU]
    -> 订单生效并创建 revision_subscription
    -> 平台记录 start_version_no / cadence / delivery_channel
    -> 新版本发布?
       -> 是:
          -> 版本发布服务生成 revision_delivery_event
          -> 若交付通道=对象包:
             -> 生成新 delivery_ticket / delivery_receipt
          -> 若交付通道=共享:
             -> 更新共享对象可见版本或 revision cursor
          -> 记录 last_delivered_version_no
          -> 若数据契约版本变更:
             -> 写入新的契约摘要与变更说明
          -> 写入账单事件与审计摘要
       -> 否:
          -> 等待下一个版本窗口
    -> 到期或停订:
       -> 停止推送
       -> 关闭 revision_subscription
```

#### 4.4.1B 只读共享交付

```text
[卖方/平台开通只读共享]
    -> 校验 recipient / subscriber / share protocol
    -> 绑定共享对象、共享范围、到期时间
    -> 生成 data_share_grant
    -> 执行 share grant:
       -> snowflake share / secure view
       -> linked dataset
       -> delta share
       -> readonly db/schema
    -> 买方首次成功读取?
       -> 是: 可触发自动验收
       -> 否: 重试共享或提出争议
    -> 到期 / 撤权:
       -> revoke share grant
       -> 写入审计与状态回执
```

#### 4.4.2 API 类交付

```text
[卖方/平台开通 API 访问]
    -> 生成 API Key / 应用绑定 / 调用配额
    -> API 网关加载授权策略
    -> 买方在有效期内调用 API
    -> 网关按调用次数、频率、IP、用途策略进行校验
    -> 返回结果
    -> 采集 access_log / usage_event / response_hash
    -> 到达最低验收条件?
       -> 是: 买方可确认验收
       -> 否: 继续订阅周期或提出质量争议
```

#### 4.4.3 沙箱 / 模板查询类交付

```text
[卖方/平台开通查询沙箱]
    -> 创建隔离执行环境
    -> 注入最小权限账号与到期时间
    -> 策略执行点控制导出、复制、网络访问
    -> 买方在沙箱内查询
    -> 采集 query_log / session_log / policy_hit / export_attempt
    -> 到期或达到次数上限
       -> 自动关闭会话
       -> 生成使用摘要
    -> 买方验收或提出争议
```

```text
[买方购买模板查询 lite SKU]
    -> 创建 template_query_grant
    -> 绑定模板摘要、输出边界、执行配额
    -> 买方提交允许参数
    -> 模板执行层校验模板白名单 / 参数范围 / 导出限制
    -> 生成受限结果对象或报告结果
    -> 写入 query_log / output_hash / policy_hit
    -> 首次成功执行后可触发自动验收
```

```text
[卖方配置查询面与模板]
    -> 为 asset_version 创建 query_surface_definition
    -> 绑定 execution_environment / storage policy / output boundary
    -> 发布 query_template_definition(versioned)
    -> 审核模板参数 schema / analysis rule / export policy
    -> 订单成交后创建 query_grant
    -> 买方执行 template_query 或 sandbox_query
    -> 生成 query_execution_run
    -> 如需导出: 生成 result_object / report_result
    -> 写入 billing_event / audit_event / fabric anchor summary
```

#### 4.4.4 报告类交付

```text
[卖方承接定制报告]
    -> 确认需求范围、交付时间、样式约束
    -> 按约输出报告文件或可视化结果
    -> 上传报告成果物与摘要
    -> 买方审核结果是否符合模板约定
    -> 符合: accept
    -> 不符合: reject / open_case
```

### 4.5 结算、退款、赔付与关闭流程

```text
[买方 accept 或争议 resolved]
    -> 订单服务判断最终责任
    -> SettlementService 生成 billing_event
    -> 结算路径:
       -> 正常成交:
          -> 释放货款给卖方
          -> 释放买卖双方保证金
       -> 卖方责任:
          -> 退款给买方
          -> 扣减卖方保证金
          -> 必要时增加赔付
       -> 买方责任:
          -> 放款给卖方
          -> 扣减买方保证金
       -> 平台责任:
          -> 平台兜底补偿或重试交付
    -> 链适配器调用 settleOrder 或 resolveCase 后续结算动作
    -> 订单状态 settled -> closed
    -> ReputationService 更新 buyer_score / seller_score
    -> AuditService 归档结算结果、责任判定、证据摘要
    -> 审计服务归档:
       -> 数据契约快照
       -> 质量报告摘要
       -> 字段结构摘要
       -> 加工责任链摘要
```

## 5. 异常与争议流程

### 5.1 自动阻断与审核前异常

```text
[商品提交审核]
    -> 命中高敏感/非法用途/地域冲突/黑名单/训练用途禁令?
       -> 是:
          -> 自动阻断
          -> 标记 risk_case
          -> 通知供方修改或终止
          -> 审计保留阻断依据
       -> 否:
          -> 进入正常审核
```

### 5.2 交付异常处理

```text
[订单处于 buyer_locked 或 seller_delivering]
    -> 超时未交付?
       -> 是:
          -> 自动触发 expireOrder 或 dispute_opened
          -> 默认卖方责任
          -> 退款 + 扣卖方保证金 + 信誉扣减
    -> 下载令牌签发失败?
       -> 是:
          -> 重试签发
          -> 超阈值后升级平台事件
    -> share grant 签发失败 / recipient 无法访问?
       -> 是:
          -> 重试 grant
          -> 超阈值后进入人工排查或争议
    -> 对象损坏/密钥封装错误?
       -> 是:
          -> 平台或卖方补交付
          -> 无法恢复则进入争议
    -> 模板查询输出越界 / 模板不可执行?
       -> 是:
          -> 立即冻结模板授权
          -> 进入争议或补交付
```

### 5.3 争议处理流程

```text
[买方 reject / 任一方 open_case]
    -> DisputeService 创建 case_id
    -> 收集 evidence:
       -> delivery_receipt
       -> download_log
       -> object_hash
       -> key_envelope
       -> contract_snapshot
       -> chat/comment/人工说明
    -> 自动裁决可处理?
       -> 是:
          -> 按规则矩阵判定责任
          -> resolveCase
          -> 进入结算/赔付
       -> 否:
          -> 流转人工复核
          -> 管理员查看证据包
          -> 形成 decision_code / penalty_code
          -> 进入结算/赔付
    -> 输出结果:
       -> refund
       -> compensation
       -> deposit_penalty
       -> reputation_delta
       -> risk_flag
```

### 5.4 保证金与信誉联动流程

```text
[订单关闭 / 争议结案 / 风险告警]
    -> ReputationService 读取:
       -> 交易评价
       -> 履约时效
       -> 争议结果
       -> 风控告警
       -> 历史稳定性
    -> 计算 reputation_next
    -> 输出新的 buyer_credit_level / seller_risk_level

    -> 若信誉过低:
       -> 提高保证金
       -> 限权
       -> 暂停上架或下单
```

## 6. 链上链下一致性与事件流

### 6.1 统一一致性主流程

```text
[应用服务接收写请求]
    -> 生成 request_id / idempotency_key
    -> 数据库事务写入:
       -> 业务对象
       -> AuditEvent
       -> outbox_event
    -> 返回业务状态 + proof_commit_state=pending
    -> publisher worker 读取 outbox_event
    -> 发布 Kafka topic
    -> 下游消费者分发至:
       -> FabricWriter
       -> AuditAnchorService
       -> NotificationService
       -> SearchIndexer
       -> RiskEngine
       -> ReportJob
    -> Fabric commit listener 回写:
       -> tx_hash
       -> proof_commit_state=confirmed/failed
    -> 外部事实回执回写:
       -> external_fact_status
    -> 对账任务周期扫描 proof / external fact / database 差异
    -> 差异存在?
       -> 是: dead letter / replay / 人工处理 / 生成对账报告
```

### 6.2 不允许出现的结果

```text
- 链上已 accepted，链下仍显示 buyer_locked
- 同一 order_id 被重复 settle
- 同一 delivery_ticket 被重复无限下载
- 数据库显示退款成功，但链上 escrow 未释放
- 同一 case_id 存在两个互斥裁决结果
```

## 6.3 搜索与索引同步流程

### 6.3.1 商品/服务索引同步

```text
[商品、SKU、标签、价格、状态、信誉、风险状态发生变化]
    -> 业务服务写 PostgreSQL 主表
    -> 同事务刷新 search.product_search_document
    -> 同事务写 outbox_event(search.product.changed)

    -> Search Indexer 消费事件
    -> 读取 PostgreSQL 最新搜索投影
    -> 组装 OpenSearch 商品/服务文档
```

```text
    -> 写 outbox_event(search.seller.changed)
    -> 发布 Kafka
    -> Search Indexer 消费并写入 seller_search_write alias
    -> 失效卖方搜索缓存
```

### 6.3.3 搜索对账与重建

```text
[搜索运维任务周期运行或人工触发]

    -> 检查 document_version / indexed_at / error 状态
    -> 发现缺失、滞后或脏文档?
       -> 否: 标记 clean
       -> 是:
          -> 创建 index_sync_task
          -> 自动补投递或 reindex
          -> 超阈值进入 dead letter / 人工处理
```

```text
       -> 更新 checkpoint(chain_confirmed)
    -> 不一致:
       -> 创建 ChainProjectionGap
       -> 进入 consistency 联查
```
