## 2. V2 流程总览图

```text
在 V1 闭环基础上新增五大主轴:

[模型商品化]
    -> [模型 API / 模型服务期商品]

[受控计算]
    -> [任务下单 -> 执行环境调度 -> 结果交付]

[多方协作]
    -> [联邦任务发起 -> 节点招募 -> 轮次训练 -> 聚合 -> 奖励]

[公链增强]
    -> [摘要锚定 -> 凭证生成 -> 公开验证]

[自动分润]
    -> [收益池形成 -> 贡献评估 -> 分润结算]

所有新增流程继续叠加:
- 合规与用途限制
- 隐私增强
- 审计证据链扩展
- 风控与异常节点治理
- 商品搜索、模型搜索与混合索引同步
- 继承 V1 的数据商品元信息、质量报告、加工责任链与数据契约快照
- 开发者调试扩展
```

## 4. 模型商品化流程

### 4.1 模型 API 商品上架流程

```text
[供方创建模型资产]
    -> 录入 model_id / model_hash / task_id / 性能指标 / 水印信息 / 授权策略
    -> 生成模型/结果商品元信息扩展:
       -> 算法来源
       -> 执行环境摘要
       -> 结果导出边界
       -> 责任与回滚条件
    -> 绑定模型/计算类数据契约模板
    -> 选择商品类型:
       -> 模型 API 商品
       -> 模型服务期商品
    -> 绑定价格、调用限制、可用用途、可导出限制
    -> 合规校验:
       -> 模型来源合法性
       -> 训练用途授权是否完备
       -> 是否允许商用
       -> 输出是否含高风险特征
    -> 风控校验:
       -> 模型污染/异常版本
       -> 重复注册/伪造哈希
    -> 审核通过
    -> 写入 ModelRegistry
    -> 可选将摘要锚定联盟链/公链
    -> 进入模型商品目录
```

### 4.2 模型 API 购买与调用流程

```text
[买方浏览模型商品]
    -> 查看能力、适用场景、指标、水印、限流与用途约束
    -> 创建订单
    -> 订单服务快照模型版本与授权策略
    -> 锁定资金 / 保证金
    -> 开通模型 API Key 或服务租约
    -> API 网关加载模型授权策略
    -> 买方调用模型 API
    -> 每次调用记录:
       -> request_id
       -> app_id
       -> model_id
       -> policy_version
       -> usage_event
    -> 到期或超额:
       -> 触发续费、超额计费或停用
    -> 形成账单与验收依据
```

## 5. 受控计算 C2D 流程

### 5.1 C2D 下单与任务编排

```text
[买方选择受控计算商品]
    -> 提交任务目标、算法模板、输入参数、期望输出
    -> 确认任务型数据契约、结果验收口径和证明要求
    -> PolicyService 校验:
       -> 用途是否允许
       -> 是否仅允许结果获取权
       -> 是否禁止原始数据导出
       -> 地域/组织/时间限制
    -> 订单服务创建 C2D 订单
    -> 锁定费用与保证金
    -> C2D Orchestrator 分配执行环境
    -> 生成 task_id / env_id
    -> 审计记录任务创建
```

### 5.2 C2D 执行与结果交付流程

```text
[执行环境启动]
    -> 拉取受控数据引用
    -> 拉取算法镜像或模板
    -> 加载策略执行点
    -> 运行任务
    -> 运行过程中:
       -> 记录 session_log
       -> 记录 policy_hit
       -> 记录 result_hash
       -> 阻断未授权导出
    -> 任务完成
    -> 输出允许结果:
       -> metrics_report
       -> aggregate_result
       -> approved_model
    -> 结果交付给买方
    -> 买方验收
    -> 结算或争议
```

## 6. 联邦学习与多方协作流程

### 6.1 联邦任务全链路

```text
[任务发起方创建 TrainingTask]
    -> 填写 model_family / rounds / min_participants / privacy_mode / reward_policy / policy_template
    -> FLTaskService 创建 task_id
    -> 合规与策略校验:
       -> allowed_regions
       -> allowed_output
       -> usage_purpose
       -> retention_days
    -> 发起节点招募
    -> 候选参与方报名
    -> 平台校验组织资质、节点信誉、在线率、算力
    -> 达到最小参与数?
       -> 否: 继续招募或终止
       -> 是: 启动任务
    -> 每一轮训练:
       -> ParticipantAgent 在本地数据上训练
       -> 仅提交梯度/更新摘要/签名/指标/可选证明
       -> AggregationService 聚合
       -> 链上提交 round_commit_hash / node_signature / timing
       -> 事件总线发送 TrainingRoundCommitted
       -> ContributionEngine 计算阶段贡献
    -> 达到 rounds / 提前停止条件
    -> 生成全局模型或结果摘要
    -> 写入 ModelRegistry / RewardContract
    -> 更新各节点信誉
    -> 若允许商品化:
       -> 形成模型商品或报告商品
```

### 6.2 隐私增强流程

```text
[训练任务启动]
    -> 选择 privacy_mode:
       -> secure_aggregation
       -> differential_privacy
       -> TEE
       -> ZKP
    -> 节点按模式执行:
       -> 安全聚合: 提交加密更新
       -> 差分隐私: 注入噪声预算
       -> TEE: 远程证明后执行
       -> ZKP: 提交执行正确性证明
    -> 策略执行点校验输出是否可导出
    -> 审计记录每轮隐私模式、证明摘要、失败原因
```

## 7. 自动分润与贡献度分润流程

### 7.1 收益形成与收益池建立

```text
[模型商品成交 / C2D 任务完成 / 联邦任务产出商品]
    -> BillingService 计算收入
    -> 按规则拆分:
       -> 平台服务费
       -> 数据提供方份额
       -> 任务发起方份额
       -> 参与节点激励池
       -> 运营或渠道份额
    -> 创建 revenue_pool
    -> 写入分润待处理事件
```

## 8. 搜索增量流程

### 8.1 模型搜索与混合检索流程

```text
[模型资产或模型商品上架/变更]
    -> 刷新 PostgreSQL search.model_search_document
    -> 写 outbox_event(search.model.changed)
    -> Search Indexer 写入 OpenSearch model_search_write alias
    -> 可选同步 embedding，用于混合召回

[买方搜索模型或受控计算商品]
    -> Search API 根据 entity_scope(product/service/model) 分流
    -> 先按全文与过滤条件召回
    -> 再结合 embedding / 热度 / 信誉 / 质量做混合排序
    -> PostgreSQL 最终校验可见性与购买资格
```

### 8.2 搜索配置与排序治理流程

```text
[搜索运营员调整 ranking_profile / synonym / autocomplete 配置]
    -> 平台保存配置版本
    -> Search Indexer 或 Search API 加载新配置
    -> 写审计与配置变更事件
    -> 必要时触发局部 reindex
```

### 7.2 分润计算与发放流程

```text
[RewardContract / ProfitShareService 接收待处理事件]
    -> 读取分润规则
    -> 读取 contribution_score:
       -> relative_metric_gain
       -> data_quality_score
       -> availability_score
       -> timeliness_score
       -> protocol_compliance_score
       -> anomaly_penalty
    -> 计算 reward_amount
    -> 风控校验:
       -> 异常节点?
       -> 串谋评分?
       -> 非法输出?
    -> 通过:
       -> 发放收益
       -> 生成收益账单
       -> 更新信誉和后续保证金
    -> 不通过:
       -> 暂停分润
       -> 发起人工复核或案件
```

## 8. 公链增强与公开验证流程

```text
[关键事件完成]
    -> 选择可公开对象:
       -> 商品摘要
       -> 订单摘要
       -> 模型摘要
       -> 凭证摘要
    -> 公链锚定服务生成摘要包
    -> 提交公链交易
    -> 返回 public_tx_hash
    -> 平台详情页展示:
       -> 凭证 ID
       -> 验证状态
       -> 浏览器链接
    -> 第三方可进行公开验证
    -> 若公链写入失败:
       -> 不影响联盟链主状态
       -> 标记 pending_anchor 并重试
```

## 9. 异常、阻断与补偿流程

### 9.1 模型训练用途规则

```text
[请求创建模型训练或训练相关任务]
    -> V2 是否受控训练路径?
       -> 否: 自动阻断
       -> 是:
          -> 检查受控环境、审计条件、输出限制
          -> 满足: 进入增强审批/人工复核
          -> 不满足: 阻断
```

### 9.2 联邦任务异常处理

```text
[训练中]
    -> 节点掉线?
       -> 记录 availability_penalty
       -> 若仍满足 min_participants，则继续
       -> 否则暂停或终止任务
    -> 上传异常更新?
       -> 本轮作废
       -> 降低贡献度
       -> 严重时冻结节点资格
    -> 聚合失败?
       -> 允许重试
       -> 超阈值后终止并进入补偿
```

### 9.3 分润异常处理

```text
[分润前风控扫描]
    -> 贡献数据缺失?
       -> 暂停分润并补采
    -> 异常主体命中?
       -> 冻结分润记录
    -> 重复分润请求?
       -> 依赖 request_id / reward_id 幂等拒绝
```

## 11. 技术架构与事件流

### 11.1 V2 事件流

```text
业务写请求
    -> 生成 request_id / idempotency_key
    -> 调用链上或链下编排器
    -> 产生事件:
       -> TrainingRoundCommitted
       -> ModelRegistered
       -> RewardCalculated
       -> RewardSettled
       -> PublicAnchorSubmitted
       -> RiskAlertRaised
    -> 消费者:
       -> ContributionEngine
       -> ProfitShareService
       -> AuditService
       -> RiskConsole
       -> PublicVerifier
```

### 11.2 新增状态机

```text
训练任务状态:
draft -> recruiting -> ready -> running -> aggregating -> completed -> published
draft/recruiting/running -> failed | aborted

模型资产状态:
draft -> registered -> reviewed -> listed -> active -> suspended -> archived

分润状态:
pending -> calculating -> risk_checking -> payable -> settled
pending/calculating/risk_checking -> frozen -> manual_review -> settled | canceled
```

## 15. 数据本体存储与信任边界分支流程

### 15.1 模型与算法的交付分支

```text
[模型/算法商品成交]
    -> 判断交付类型
       -> 模型文件交付
          -> 读取 artifact_storage_mode / weight_custody_mode
          -> 平台托管密文模型或第三方模型仓库交付
       -> 模型服务调用
          -> 平台只签发服务权限和计费策略
          -> 买方不直接获取权重
       -> 受控训练结果
          -> 平台交付结果、摘要、证明，不交付原始训练语料
```

### 15.2 受控执行与训练数据不出域

```text
[创建 compute/training task]
    -> 校验 training_data_residency_mode
    -> 平台编排任务与证明
    -> 数据保留在卖方侧或受控执行环境
    -> 执行环境返回 result_hash / proof_hash / result_summary
    -> 平台完成结果审查、导出审查、账单和分润
```

### 15.3 结果导出边界

```text
[申请导出结果]
    -> 读取 result_export_policy
    -> 校验证明材料、环境状态、合同与策略
    -> 允许:
       -> 生成结果交付记录和证据
    -> 不允许:
       -> 阻断导出
       -> 写风险与审计事件
```

