# 数据产品分类与交易模式详细稿

- 文档名称：数据产品分类与交易模式详细稿
- 适用范围：基于区块链技术的数据交易平台 V1 / V2 / V3
- 版本：D1
- 日期：2026-04-07
- 上位输入：PRD-R4《基于区块链技术的数据交易平台需求清单（需求基线增强整合版）》

---

## 1. 文档目标

本文件用于把 PRD 中“交易对象、SKU、权利模型、交付方式、验收方式”进一步收敛为产品建模与交易建模基线，主要回答：

1. 平台卖的到底是什么。
2. 不同数据产品分别适合什么交易模式。
3. 哪些权利可卖、哪些默认不可卖。
4. 不同产品类型的计费、交付、验收、退款如何区分。
5. V1 应该先支持哪些标准 SKU，V2/V3 再扩展哪些高级模式。

---

## 2. 基础定义

## 2.1 四层模型

平台产品建模按以下四层进行：

1. **Data Resource**：原始资源层，记录来源、范围、质量、法律属性。
2. **Data Product**：对外可展示、可定价、可审核的产品层。
3. **Product SKU**：具体可下单的权利包层。
4. **Contract + Usage Policy**：将 SKU 在具体交易中实例化为可执行授权。

## 2.2 产品不是“文件”，而是“权利 + 交付 + 限制”的组合

一个 SKU 必须同时回答：
- 卖的是哪种权利
- 通过什么方式交付
- 可以怎样用
- 不可以怎样用
- 如何计费
- 如何验收
- 如何退款/赔付

## 4. 权利模型

## 4.1 权利集合

- 使用权
- 访问权
- 查询权
- 计算权
- 结果获取权
- 内部共享权
- 模型训练权
- 商业输出权
- 转授权权

## 4.2 V1 权利边界

V1 正式支持：
- 使用权
- 访问权
- 查询权
- 结果获取权
- 有限内部共享权

V1 默认禁止：
- 所有权转让
- 无限期模型训练权
- 转授权权
- 再分发权
- 无约束商业输出权

## 4.3 V2 扩展权利

- 计算权
- 受控模型训练权
- 结果级商业输出权
- 多方协作训练权
- 受限证明验证权

## 5. 产品类型 × 交易模式总览

| 产品类型 | 分阶段归属 | 主要交易模式 | 默认权利 | 默认交付 | 默认验收 | 默认计费 |
|---|---|---|---|---|---|---|
| 文件副本 | V1 | 直接购买、分批交付 | 使用权 | 下载 | 哈希校验 + 下载成功 | 一次性/分批 |
| API / 订阅 | V1 | 月订阅、按量计费 | 访问权、结果获取权 | API Key / OAuth | 首次成功调用 | 基础订阅 + 超额 |
| 查询沙箱 | V1 | 席位订阅、按次查询 | 查询权、结果获取权 | 沙箱账号 | 模板可用 + 首次成功查询 | 月费/按次 |
| 固定报告 | V1 | 定制报价、单次交付 | 结果获取权 | 报告下载/发送 | 报告签收 | 单次/阶段 |
| 模型 API | V2 | 调用量订阅 | 访问权、结果获取权 | 模型 API | 首次成功调用 | 调用量/服务期 |
| C2D 任务 | V2 | 按任务购买 | 计算权、结果获取权 | 任务执行 | 结果模板匹配 | 任务费/算力费 |
| 联邦学习 | V2 | 项目制协作 | 参与训练权、结果获取权 | 联合训练任务 | 阶段验收 | 项目费 |
| MPC 结果 | V2 | 按任务购买 | 结果获取权 | 多方计算输出 | 结果生成成功 | 按任务 |
| ZKP 证明 | V2 | 按次证明 | 证明验证权、结果获取权 | 证明回执 | 验签成功 | 按次 |

## 6. V1 标准数据产品目录

## 6.1 文件副本产品

### 适用场景
- 工业质量日报
- 周/月经营汇总包
- 标签包
- 合成数据包
- 低敏报表附件

### 默认规则
- 仅卖使用权，不卖所有权
- 必须带版本号
- 必须带水印或指纹
- 默认禁止再次分发与训练

### 推荐数据等级
- P0 / P1 / P2
- P3 仅在脱敏充分且审批通过时考虑

## 6.2 API / 流式订阅产品

### 适用场景
- 设备运行指标
- 门店经营指标
- 商圈画像查询
- 流量、库存、销量指标

### 默认规则
- 交付单位是 Application 级访问权
- 默认需要配额、QPS、IP 限制
- 默认禁止明细拉平形成副本库

### 推荐数据等级
- P1 / P2 / 部分 P3 聚合结果

## 6.3 查询沙箱产品

### 适用场景
- 供应链协同查询
- 选址分析
- 指标钻取
- 经营诊断

### 默认规则
- 默认按席位授予
- 默认限制结果行数、字段范围、导出次数
- 默认禁止原始明细表导出

### 推荐数据等级
- P2 / P3

## 6.4 固定报告产品

### 适用场景
- 月报/季报
- 市场洞察报告
- 选址诊断报告
- 风险评估报告

### 默认规则
- 报告视为结果产品
- 默认不开放底层数据
- 默认以签收/确认作为交付依据

---

## 7. V2 扩展产品目录

## 7.1 C2D 计算权产品
- 算法进场
- 数据不出域
- 输出模板受限
- 强依赖白名单算法、执行环境与证明材料

## 7.2 联邦学习产品
- 多方参与训练
- 输出为聚合模型、指标或报告
- 必须有参与方管理、轮次管理、聚合策略

## 7.3 MPC 结果产品
- 多方输入共同计算
- 只交付结果
- 不允许看到中间输入

## 7.4 ZKP 证明产品
- 只交付证明与验证结果
- 适合资格/合规/年龄/信用事实证明

## 7.5 模型 / 算法服务产品
- 预测评分 API
- 画像分析 API
- 推荐/匹配能力 API
- 默认卖能力，不卖权重
- 默认禁止逆向与权重复制
- 商业输出权需额外审批

## 8. 标准 SKU 体系

## 8.1 文件类 SKU

### SKU-F1：文件单次下载包
- 权利：使用权
- 交付：一次性下载令牌
- 有效期：7 / 30 / 90 天
- 验收：下载成功 + 哈希一致
- 定价：单次固定价

### SKU-F2：文件分批交付包
- 权利：使用权
- 交付：按周/月版本包
- 验收：每批次独立验收
- 定价：总包价 / 阶段价

## 8.2 API 类 SKU

### SKU-A1：API 基础订阅包
- 权利：访问权
- 交付：Application 绑定 + Key
- 配额：固定调用量
- 验收：首调成功
- 定价：月费

### SKU-A2：API 基础订阅 + 超额计费
- 权利：访问权 + 结果获取权
- 交付：与 A1 相同
- 配额：超额后继续服务
- 验收：首调成功
- 定价：月费 + 按量

### SKU-A3：API 阶梯档位包
- 适合高频工业或零售指标
- 根据 QPS / 调用量档位计费

## 8.3 沙箱类 SKU

### SKU-S1：单席位月包
- 权利：查询权
- 交付：用户席位
- 验收：首次登录 + 样例查询成功
- 定价：月费

### SKU-S2：多席位团队包
- 权利：查询权 + 内部共享权（部门内）
- 交付：多个用户席位
- 验收：席位全部生效
- 定价：阶梯席位价

### SKU-S3：按次查询包
- 权利：限定查询权
- 交付：查询次数额度
- 验收：首个查询成功
- 定价：按次

## 8.4 报告类 SKU

### SKU-R1：标准报告包
- 权利：结果获取权
- 交付：报告 PDF / 下载链接
- 验收：签收或超时自动验收
- 定价：单份固定价

### SKU-R2：定制报告项目包
- 权利：结果获取权
- 交付：阶段性报告
- 验收：按里程碑
- 定价：项目价

## 8.5 V2 模型类 SKU

### SKU-M1：模型 API 调用包（V2）
- 权利：访问权
- 交付：模型 API
- 验收：首调成功
- 定价：月费 / 调用量

### SKU-M2：模型服务期包（V2）
- 权利：访问权 + 受限商业输出权（审批后）
- 定价：服务期固定价 + 调用量附加

## 8.6 V2 计算类 SKU

### SKU-C1：受控计算任务包
- 权利：计算权 + 结果获取权
- 交付：任务执行权
- 验收：任务成功 + 输出模板匹配
- 定价：任务费 + 算力费

## 9. 交易模式定义

## 9.1 目录直购
适用：
- 标准文件包
- 标准 API 订阅
- 标准沙箱包

特点：
- 价格清晰
- 适合 V1 主路径
- 合同模板标准化

## 9.2 RFQ / 人工报价
适用：
- 定制报告
- 非标准 API
- 大客户包年

特点：
- 由买方发起需求
- 平台或供方手动报价
- 适合复杂条款

## 9.3 周期订阅
适用：
- API
- 沙箱
- 定期报告

特点：
- 自动续费或到期断权
- 支持基础订阅 + 超额用量

## 9.4 按次购买
适用：
- 单份文件
- 单次查询
- 单次报告
- 单次证明服务

## 9.5 项目制
适用：
- 联邦学习
- 定制数据服务
- 长周期建模协作

## 9.6 KPI 展示型公链交易凭证
适用：
- V2 公链展示层
- 已完成交易或认证事件
- 不作为核心交易模式，只作为展示或外部验证层

## 11. 产品类型 × 风险点矩阵

| 产品类型 | 主要风险 | 默认控制 |
|---|---|---|
| 文件副本 | 复制扩散、再分发、训练滥用 | 水印、限次下载、用途限制 |
| API | 凭证共享、批量爬取、过量调用 | Application 绑定、配额、限流、IP 白名单 |
| 沙箱 | 结果逆向识别、模板逃逸、导出滥用 | 导出限制、模板白名单、审计留痕 |
| 报告 | 结果误导、描述不符 | 明确口径、签收留痕 |
| 模型 API | 逆向、滥用、结果误用 | 调用审计、输出边界、合同限制 |
| C2D | 算法外传、输出越界 | 算法白名单、输出模板、证明材料 |
| 联邦学习 | 模型投毒、参数泄露 | 参与方审查、轮次审计、聚合控制 |
| MPC | 结果操纵、任务失败 | 多方授权、结果校验 |
| ZKP | 错误证明或假证明 | 验签、证明链路审计 |

## 12. 产品与行业映射（首批场景）

## 12.1 工业制造 / 供应链

优先产品：
- API 指标订阅
- 质量日报文件包
- 供应链查询沙箱
- 模型评分 API（可选）

不优先：
- 高敏原始日志全量下载
- 联邦训练正式生产交付（V1 不做）

## 12.2 零售 / 本地生活

优先产品：
- 门店经营 API
- 商圈画像查询沙箱
- 经营分析报告
- 选址分析按次查询

不优先：
- 含敏感个人信息明细下载
- 跨平台联合训练正式交付（V1 不做）

## 13. 不允许的产品组合

V1 禁止以下组合：

1. 文件副本 + 模型训练权
2. 文件副本 + 转授权权
3. 境外主体 + 下载型高敏 SKU
4. 自然人供方 + 敏感个人信息副本
5. 沙箱产品 + 原始明细导出
6. API 产品 + 不限量明细全量拉取
7. 公开链展示 + 原始合同正文 / 原始身份信息

## 16. 结论

这份产品分类与交易模式设计的核心是把“卖数据”明确重构成：

- 卖文件包
- 卖访问权
- 卖查询权
- 卖结果
- 卖能力
- 卖计算权

而不是一个模糊的“卖数据集”概念。  
V1 先把文件、API、沙箱、报告四类产品做标准化闭环；V2 再把模型服务、受控计算、多方协作和证明型产品纳入正式交易体系。  
能力范围不删减，只做分阶段实现：V1 不承诺模型类商品正式上线，V2 起进入完整商品化与合规控制。

## 17. 支付、收费与结算绑定补充

### 17.1 产品不是只绑定价格，还必须绑定支付与收费规则

每个商品 / SKU 除了绑定合同模板、验收模板、退款模板、计费模板，还必须绑定：

- 支付方式白名单
- 起步司法辖区快照
- 允许的付款币种 / 收款币种
- 走廊策略
- 支付时点
- 是否要求买方保证金
- 是否要求卖方履约保证金
- 平台服务费规则
- 渠道手续费承担方
- 是否允许周期结算
- 是否允许线下对公转账

### 17.2 V1 推荐绑定方式

- 文件/报告：一次性支付意图 + 一次性结算。
- API：周期支付 + 超额账单事件。
- 沙箱：席位/周期支付 + 结果导出附加收费。
- RFQ/项目制：允许人工报价，但支付与费用快照必须冻结到订单。

### 17.3 后续能力

- `V2`：商品可绑定自动分润规则、周期扣费策略和打款策略。
- `V3`：商品可绑定多币种/跨境/数字资产结算路由，但不得反向影响 V1 商品骨架。

## 18. 新增：商品交付与身份认证绑定规则

- 文件副本商品：交付前必须完成登录、会话校验、下载令牌校验；高敏下载可要求 step-up。
- API / 订阅商品：必须绑定 `Application`、令牌或 OAuth/OIDC 授权，不允许共享凭证。
- 沙箱商品：必须绑定成员身份、会话和设备信任，不允许共享登录。
- 模型与受控计算商品：必须绑定调用主体、任务主体和必要的服务身份；是否允许导出结果由身份等级和策略共同决定。
- 链上签名、链上回执、联盟链成员动作必须通过已绑定的 Fabric 身份完成，不得直接复用普通网页登录状态。
