# 合规审查与风控规则详细稿

- 文档名称：合规审查与风控规则详细稿
- 适用范围：基于区块链技术的数据交易平台 V1 / V2
- 版本：D1
- 日期：2026-04-07
- 上位输入：PRD-R4《基于区块链技术的数据交易平台需求清单（需求基线增强整合版）》

---

## 1. 文档目标

本文件用于把 PRD 中的“合规能力、风控规则、高风险清单、威胁模型”落成可进入规则引擎、审核工作台、工单系统的需求基线，主要回答：

1. 平台在注册、上架、下单、交付、使用、结算、争议阶段分别审什么。
2. 哪些规则必须自动判断，哪些必须人工复核。
3. 高风险事项如何分层审批。
4. 命中违规或风险后，系统应采取什么动作。
5. V1 应落地到什么粒度，V2 再扩展哪些能力。

---

## 2. 适用原则

1. 合规先于交易，交易先于交付，交付先于持续使用。  
2. 所有审查必须结构化输入，禁止仅靠自由文本判断。  
3. 高风险事项必须能被规则引擎识别并阻断。  
4. 系统做规则判断，人工做例外判断与责任确认。  
5. 任何结论都必须留痕，可追溯到输入字段、命中规则、审批人、证据材料。  
6. 风控覆盖交易前、交易中、交易后，不得只在注册阶段做一次性校验。

## 3. 审查对象与时间点

## 3.1 主体准入审查
对象：
- 卖方主体
- 买方主体
- 算法/模型服务方
- 连接器/执行环境服务方

时点：
- 注册前
- 注册后升级认证
- 高风险能力开通前

## 3.2 产品上架审查
对象：
- Data Resource
- Data Product
- Product SKU
- 样例与质量报告
- 来源与权利证明

时点：
- 首次上架
- 重大变更
- 版本升级

## 3.3 订单与授权审查
对象：
- 订单
- 合同
- Usage Policy
- 付款与交付路径
- 使用目的与输出边界

时点：
- 下单前
- 合同生效前
- 扩权前

## 3.4 持续使用审查
对象：
- API 调用
- 沙箱查询
- 导出行为
- 扩权申请
- 续费续权
- 公链展示开关

时点：
- 交易中
- 交易后持续监管

## 3.5 争议与应急审查
对象：
- 退款申请
- 赔付申请
- 合规变更
- 产品撤回
- 主体解冻/恢复

## 5. 风险分类框架

## 5.1 风险类型

1. 法律合规风险
2. 数据权属风险
3. 隐私风险
4. 商业秘密风险
5. 安全风险
6. 履约风险
7. 财务风险
8. 声誉风险
9. 公链披露风险

## 5.2 风险级别

- **L0 低风险**：可自动放行
- **L1 一般风险**：自动放行但记日志
- **L2 中风险**：自动放行 + 后置抽查
- **L3 高风险**：人工复核
- **L4 极高风险**：专项审批或默认阻断
- **L5 禁止**：直接阻断

## 5.3 风险来源维度

- 主体风险
- 数据风险
- 用途风险
- 行为风险
- 履约风险
- 跨链/展示风险

## 6. 禁止 / 限制 / 需审批清单

## 6.1 禁止清单（直接阻断）

1. 原始敏感个人信息副本交易
2. 医疗/金融高敏明细直接下载
3. 无权利证明或来源证明的数据上架
4. 自然人自由上架敏感个人信息副本
5. 境外主体申请下载型高敏产品
6. 未说明用途却申请模型训练
7. 合同模板与 SKU 类型不匹配
8. 公链展示原始数据、实名身份、合同正文、敏感日志

## 6.2 限制清单（仅特定条件可做）

1. 含个人信息的聚合结果产品
2. 公共数据授权运营产品
3. 重要数据相关产品
4. 商业输出权申请
5. 模型 API 作为对外服务能力再出售
6. 公链展示开关启用

## 6.3 需审批清单

1. 高金额订单
2. 高风险行业订单
3. 境内高敏数据受控计算
4. 扩权申请
5. 导出范围变更
6. 结果级商业输出
7. 公链凭证批量签发

## 7. 合规审查状态机

## 7.1 主体审查状态
`draft -> submitted -> auto_check_pass / auto_check_fail -> manual_review -> approved / rejected / restricted`

## 7.2 产品审查状态
`editing -> submitted -> auto_check -> manual_review_if_needed -> approved / rejected / needs_fix -> published`

## 7.3 订单审查状态
`created -> rule_check -> blocked / manual_review / approvable -> contract_pending -> approved / rejected`

## 7.4 交易中风控状态
`running -> risk_hit -> alert / throttle / freeze_partial / freeze_full -> resolved / escalated`

## 10. 审批层级定义

- **L1**：系统自动判断，无需人工参与
- **L2**：平台单角色人工复核
- **L3**：双角色复核（如合规 + 法务）
- **L4**：专项审批（高风险 / 重要数据 / 公共数据 / 高敏）
- **L5**：平台治理层/监管协同场景

V1 最小要求：
- 至少支持 L1 / L2 / L3
- L4 先以人工会签或工单方式实现
- L5 不做完整自动流转

## 13. 争议与合规联动规则

出现以下任一情况，必须进入争议或应急合规模块：

1. 产品描述与交付不符
2. 权利证明被撤销或失效
3. 数据上游被认定违规采集
4. 使用行为超出合同边界
5. 账单计费异常
6. 公链展示与链下真实状态不一致

处置路径：
- 冻结相关金额
- 冻结新购/续费
- 评估是否停存量访问
- 输出证据包
- 更新信用记录
- 必要时撤销公链展示或吊销凭证

## 14. 系统安全控制基线

## 14.1 身份与认证
- 登录必须支持 MFA（高风险角色）
- 高风险动作需二次确认
- 服务身份单独治理

## 14.2 数据与密钥
- 链下敏感数据静态加密
- 下载令牌短时有效
- 密钥、令牌、证书可轮换可吊销

## 14.3 日志与审计
- 审计日志与业务日志隔离
- 关键证据日志防篡改
- 访问、下载、调用、导出、冻结、解冻必须全量留痕

## 14.4 运行与环境
- 连接器、容器、应用需安全扫描
- 安全事件进入 SOC/SIEM
- 灾备与恢复演练纳入计划

---

## 15. 公链展示合规规则

1. V1 生产默认关闭公链展示。
2. 仅允许联盟链事件批次哈希同步到测试网或演示环境。
3. 公开链只能展示摘要、批次、公开元数据。
4. 不得展示原始数据、实名身份、合同正文、敏感日志。
5. 供应商认证凭证、交易凭证、护照 NFT 必须支持吊销和到期失效。
6. 公链展示必须有单独开关、披露级别与 Tenant 级禁用能力。

## 18. 结论

这套合规与风控设计的关键不是把规则写成“原则宣言”，而是把它拆成：

- 输入字段
- 命中条件
- 系统动作
- 人工角色
- 阻断点
- 证据材料
- 时效要求

只有做到这一层，平台才能真正从“有人审核”升级到“系统可执行、人工可复核、全程可追溯”的合规与风控体系。

## 19. 支付、资金流与清结算风控规则补充

### 19.1 必须额外结构化采集的字段

- `payment_provider`
- `payment_method`
- `payment_intent_type`
- `payer_subject`
- `payee_subject`
- `channel_fee_bearer`
- `fee_rule_version`
- `deposit_required`
- `payout_mode`
- `reconciliation_status`

### 19.2 自动阻断类规则

- `V1` 生产环境未命中新加坡起步司法辖区与走廊白名单时，不得发起真实支付。
- 支付成功前不得进入交付。
- 未通过回调验签的支付结果不得更新订单状态。
- 重复 webhook 不得导致重复记账、重复退款或重复打款。
- 账务镜像与支付结果不一致时，不得自动进入结算完成。
- 未完成对账差异处置的高金额订单不得自动打款。

### 19.3 人工复核类规则

- 新加坡起步走廊以外但拟进入后续灰度的支付请求
- 异常大额退款
- 异常高频退款
- 打款失败后重复执行
- 渠道账单与平台账务不一致
- 涉及保证金扣罚与赔付联动的支付结果

### 19.4 V1 / V2 / V3 边界

- `V1`：支付回调幂等、人工退款、人工打款、差异工单。
- `V2`：自动打款、分润结果复核、周期扣费风控。
- `V3`：多币种、跨境和数字资产结算纳入高级风控。

### 20.2 自动阻断类规则

- 被冻结主体或成员不得登录
- 未启用 MFA 的高风险角色不得执行高风险动作
- 证书已吊销、已过期或状态异常的链上身份不得继续执行链上签名
- 风险设备或高风险国家/地区异常登录未完成 step-up 前不得进入支付、退款、证书治理链路

### 20.3 人工复核类规则

- 企业 SSO 首次接入后的高权限成员自动建档
- Fabric 身份首次签发
- Fabric 属性模板变更
- 高风险角色 MFA 解绑
- 可疑设备信任申请
