# 数据原样处理与产品化加工流程设计

- 文档名称：数据原样处理与产品化加工流程设计
- 适用范围：基于区块链技术的数据交易平台 `V1 / V2 / V3`
- 版本：D1
- 日期：2026-04-10
- 上位输入：`原始PRD/基于区块链技术的数据交易平台-需求清单-PRD-R1.md`
- 关联专题稿：
  - `原始PRD/数据本体存储与信任边界设计.md`
  - `原始PRD/数据商品元信息与数据契约设计.md`
  - `原始PRD/双层权威模型与链上链下一致性设计.md`

---

## 1. 文档目标

本设计用于回答五个问题：

1. 平台如何把“原始数据”变成“可卖的数据产品”。
2. 不同数据类型在交易前后分别应走什么加工链路。
3. 哪些处理必须保留原始格式，哪些处理需要类内标准化。
4. 哪些对象需要落库，哪些对象只保留对象存储与摘要。
5. 这些流程如何并入现有的商品、交付、验收、审计、上链和支付闭环。

本文件的核心结论是：

**平台不应把所有数据转成同一种格式，而应采用“原始格式保留 + 类内标准化 + 统一治理层”的多形态加工模型。**

---

## 2. 核心设计原则

### 2.1 原始格式必须保留

平台对原始接入对象的第一原则不是“立刻改写”，而是：

- 原始对象先登记
- 原始对象先生成指纹和来源证明
- 原始对象先进入受控接入区
- 原始对象在进入类内标准化前必须可回放、可对账、可追责

### 2.2 统一治理的是治理层，不是原始编码格式

统一的对象包括：

- 元信息档案
- 字段结构说明
- 质量报告
- 加工责任链
- 数据契约
- 版本与 revision
- 授权、交付、验收、撤权
- 审计证据与链上锚定

不要求统一的对象包括：

- CSV、Excel、Parquet、JSON、PDF、图片、音视频、数据库 dump、模型文件本身

### 2.3 平台内处理能力是可选能力，不是默认明文接管

平台可以提供：

- 清洗
- 标准化
- 脱敏
- 标注
- 摘要提取
- 预览生成
- 质量规则执行
- 产品包装

但必须显式声明：

- 是平台直接处理
- 是卖方侧处理后上传结果
- 是第三方加工方处理
- 还是仅登记外部处理结果与证据

### 2.4 文件快照、共享、API、模板查询、结果产品必须走不同后处理规则

它们至少在以下方面不同：

- 交付对象
- 验收规则
- 撤权方式
- 到期方式
- 退款与赔付口径
- 审计证据范围

## 3. 六区模型

平台正式采用六区模型：

1. 原始接入区 `Raw Ingress`
2. 分类识别区 `Classify`
3. 类内标准化区 `Normalize`
4. 加工处理区 `Process`
5. 产品包装区 `Productize`
6. 交付运营区 `Deliver`

### 3.1 原始接入区

职责：

- 接收原始对象
- 记录来源、上传方、批次、时间
- 生成对象摘要和指纹
- 执行杀毒、文件安全、类型识别预检查
- 决定是否允许进入后续加工

### 3.2 分类识别区

职责：

- 识别对象类型
- 识别编码与容器格式
- 判断处理路径
- 决定是否需要类内标准化

平台至少支持：

- 表格/关系型数据
- 半结构化数据
- 文档文本类
- 图片
- 音频/视频
- 数据库快照/转储
- API/服务商品
- 模型/代码/Notebook

### 3.3 类内标准化区

职责：

- 对需要结构理解的对象做类别内标准化
- 生成本类别统一可治理结构

例如：

- 表格类：统一成 schema + 字段类型 + 主键/时间字段 + 采样统计
- JSON 类：统一成 path schema + 枚举/缺失统计
- 文档类：统一成提取文本 + 章节索引 + 摘要
- 图片/音视频：统一成 sidecar manifest + 标签/标注/预览

### 3.4 加工处理区

职责：

- 清洗
- 标准化
- 去重
- 脱敏
- 标注
- 切片/转码
- 摘要提取
- 特征提取
- 样本与预览生成

### 3.5 产品包装区

职责：

- 形成 `DataAsset`
- 形成 `AssetVersion`
- 形成 `AssetObjectBinding`
- 形成元信息档案
- 形成字段结构说明
- 形成质量报告
- 形成加工责任链
- 形成数据契约
- 形成样例与预览对象

### 3.6 交付运营区

职责：

- 上架审核
- 交易交付
- 授权开通
- 验收结算
- 撤权过期
- 审计归档
- Fabric 摘要锚定

## 4. 交易前正式流程

交易前正式流程建议固定为 12 步：

1. 主体准入与资质认证
2. 原始接入登记
3. 格式识别与对象分类
4. 权利链登记
5. 分类分级
6. 初步质量画像
7. 预处理决策
8. 类内标准化
9. 加工处理执行
10. 产品包装
11. 版本化与 revision 治理
12. 上架审核与可交付性证明

### 4.1 主体准入与资质认证

输出：

- `seller_org_id`
- `license_snapshot`
- `identity_proof_snapshot`
- `baseline_risk_level`

### 4.2 原始接入登记

输出：

- `raw_ingest_batch`
- `raw_object_manifest`
- `raw_object_hash`
- `source_declared_rights`

### 4.3 格式识别与对象分类

输出：

- `detected_object_family`
- `mime_type`
- `container_type`
- `classification_confidence`
- `recommended_processing_path`

### 4.4 权利链登记

输出：

- 所有者
- 采集者
- 加工者
- 第三方来源
- 授权来源
- 权利限制来源

### 4.5 分类分级

输出：

- 公开 / 内部 / 敏感 / 高敏
- `contains_pi / contains_spi / contains_important_data`
- 商业秘密标记
- 跨域与行业限制标记

### 4.6 初步质量画像

输出：

- 行数 / 文件数 / 页数 / 时长 / 分辨率
- 字段数 / 表数 / 章节数
- 缺失率 / 异常率 / 覆盖率
- 时间覆盖范围
- 更新时间与刷新频率

### 4.7 预处理决策

平台必须在这一阶段明确处理模式：

- 平台托管直交付
- 平台托管但加密交付
- 卖方自持交付
- 受控执行交付
- 第三方可信存储交付

### 4.8 类内标准化

不是把所有数据转成同一种格式，而是转到本类别规范格式：

- 表格/关系型：`schema + row stats + parquet/csv/table binding`
- 半结构化：`json schema + path stats + jsonb metadata`
- 文档：`text extract + chapter index + summary`
- 图片/音视频：`sidecar manifest + preview + labels`
- 数据库快照：`schema inventory + sample stats`
- 模型/代码：`artifact manifest + dependency lock + model card`

### 4.9 加工处理执行

可执行任务包括：

- 去重
- 缺失修复
- 字段映射
- 单位标准化
- 脱敏
- 标注
- OCR / ASR
- 转码 / 切片
- 特征提取
- 摘要提取

### 4.10 产品包装

形成的正式对象至少包括：

- `asset`
- `asset_version`
- `asset_object_binding`
- `product`
- `product_sku`
- `metadata_profile`
- `field_definition`
- `quality_report`
- `processing_job`
- `data_contract`
- `sample / preview / sidecar manifest`

### 4.11 版本化与 revision 治理

必须支持：

- 版本号
- 发布说明
- 历史版本保留策略
- 未来 revision 推送策略
- 订阅策略
- 回滚与作废策略

### 4.12 上架审核与可交付性证明

审核内容至少包括：

- 权利链
- 分类分级
- 质量报告
- 数据契约
- 交付方式
- 定价
- 验收规则
- revoke 可执行性

可交付性证明包括：

- 托管对象加密上传成功
- 样例与预览可访问
- 共享对象可授权/可撤权
- API 健康检查通过
- 模板查询白名单与输出边界生效

## 5. 交易后正式流程

交易后正式流程建议固定为 9 步：

1. 订单成立与资金锁定
2. 买方预验收
3. 授权开通或正式交付
4. 验收执行
5. 最终结算
6. 分润结算
7. 撤权与过期控制
8. 审计归档
9. Fabric 摘要锚定

### 5.1 买方预验收

买方预验收阶段只允许查看：

- schema / 字段结构摘要
- sample / preview
- 质量报告
- 数据契约
- 试算结果或小样本结果

默认不允许先拿全量原始数据。

### 5.2 授权开通或正式交付

根据商品类型分流：

- 文件快照：下载票据 + 密钥封装
- 只读共享：recipient grant + revoke hook
- API：token / client credential + quota
- 模板查询：grant + template whitelist + parameter policy
- 结果产品：结果包或报告交付

### 5.3 撤权与过期控制

必须支持：

- token 失效
- 共享 revoke
- 订阅停止
- 模板下线
- 结果到期不可再取

## 6. 不同类型数据的处理策略

## 6.1 表格/关系型数据

原始保留：

- `csv`
- `xlsx`
- `sql dump`
- 原始表/视图导出

标准化输出：

- `schema json`
- 字段类型
- 主键/时间字段
- 采样统计
- `parquet / csv / table/view binding`

## 6.2 半结构化数据

原始保留：

- `json`
- `jsonl`
- `xml`
- `avro`

标准化输出：

- path schema
- 枚举值摘要
- 可选字段统计
- `jsonb metadata`

## 6.3 文档文本类

原始保留：

- `pdf`
- `doc/docx`
- 扫描件

标准化输出：

- 提取文本
- 页级/章节级索引
- 关键词摘要
- 预览图

## 6.4 图片

原始保留：

- 原图对象

标准化输出：

- `sidecar manifest`
- 尺寸/编码信息
- 预览图
- 标签文件
- 标注文件

## 6.5 音频/视频

原始保留：

- 原始音视频对象

标准化输出：

- 转码信息
- 切片信息
- 字幕或 `ASR` 文本
- 标签与摘要

## 6.6 数据库快照/转储

原始保留：

- dump
- binlog
- export package

标准化输出：

- schema 描述
- 表清单
- 行数与采样统计
- 共享对象定义

## 6.7 模型/代码/Notebook

原始保留：

- 模型权重
- Notebook
- 镜像
- 代码仓导出包

标准化输出：

- artifact manifest
- 依赖锁定
- model card
- 输入输出契约
- 运行环境摘要

## 7. 数据库与对象模型落点

现有对象继续沿用：

- `catalog.data_asset`
- `catalog.asset_version`
- `catalog.asset_object_binding`
- `catalog.product`
- `catalog.product_sku`
- `catalog.product_metadata_profile`
- `catalog.asset_field_definition`
- `catalog.asset_quality_report`
- `catalog.asset_processing_job`
- `catalog.asset_processing_input`
- `contract.data_contract`

本专题新增建议对象：

- `catalog.raw_ingest_batch`
- `catalog.raw_object_manifest`
- `catalog.format_detection_result`
- `catalog.extraction_job`
- `catalog.preview_artifact`

其中：

- 原始文件本体、转码结果、大体量中间产物仍放对象存储
- PostgreSQL 保存批次、摘要、类型、任务状态、版本关系、责任链与审计引用

至少新增一组供方页面：

- 原始接入中心
- 原始对象详情页
- 分类识别与标准化页
- 加工任务页
- 样例与预览页

- 原始接入创建/查看/管理

- 平台审核员
- 平台审计安全员

## 9. 阶段策略

### 9.1 V1

必须实现：

- 原始接入登记
- 分类识别
- 基础类内标准化
- 基础加工责任链
- 样例与预览对象
- 数据契约与质量报告绑定

可暂不实现：

- 高级多媒体特征抽取
- 复杂 OCR/ASR 管线
- 自动化图像/音视频高级标注

### 9.2 V2

补齐：

- 模型、代码、Notebook 制品包装
- 受控计算输入输出包装
- 更复杂的 extraction / transform / labeling 工具链

### 9.3 V3

补齐：

- 跨平台原始对象引用
- 外部加工责任链互认
- 外部质量证明与外部 preview 引用

---

## 10. 一句话结论

这套设计把平台的数据加工能力收口为：

**原始接入可追责、分类识别可决策、类内标准化可治理、加工责任链可审计、产品包装可交易、交付撤权可回放。**
