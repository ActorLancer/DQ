## 3. 全局页面规范

### 3.1 新增对象主键

- `ccr_id`
- `ack_hash`
- `alert_id`
- `freeze_ticket_id`
- `governance_case_id`
- `regulator_query_id`
- `connector_id`
- `partner_id`

### 3.2 新增状态标签

- 跨链请求：`initiated / signed / relayed / acknowledged / finalized / failed`
- 冻结工单：`opened / approved / executed / observing / released / archived`
- 风险工单：`new / triaged / investigating / actioned / closed`

### 3.3 权限设计引用

本文件页面权限以以下文档为准：

- [权限点清单.md](/home/luna/Documents/DataB/权限设计/权限点清单.md)
- [角色权限矩阵正式版.md](/home/luna/Documents/DataB/权限设计/角色权限矩阵正式版.md)
- [菜单权限映射表.md](/home/luna/Documents/DataB/权限设计/菜单权限映射表.md)

新增页面实现要求：

- 跨链、风控、监管、治理、连接器页面必须区分只读和治理权限
- 终止跨链、补偿、冻结、解冻、监管证明导出、互认关系变更属于高风险操作

### 3.4 搜索增量要求

- 在 V1/V2 搜索对象基础上新增合作伙伴、连接器与跨平台卖方搜索
- 搜索结果需叠加互认状态、监管限制和司法辖区过滤
- 生态搜索索引仍然属于 OpenSearch 读模型，交易与可见性判断仍回 PostgreSQL

### 3.5 跨平台元信息与数据契约要求

- 跨平台商品详情页、互认关系页和监管穿透页必须展示契约摘要、责任边界摘要和外部质量证明摘要。
- 对端平台提供的字段说明、质量证明和责任链信息需要与本平台自声明信息分开展示。
- 跨域对象缺失契约摘要或责任边界摘要时，不得进入正式互认与交易态。

## 4. 跨链治理页面

### 4.1 跨链治理工作台

**页面目标**

- 汇总所有跨链请求、同步类型、目标链、回执和异常状态

**核心模块**

- 请求列表
- 状态筛选
- 重试/终止统计
- 目标链分布
- 风险标记

### 4.2 跨链请求详情页

**页面目标**

- 查看单个 `CrossChainRequest` 的完整治理生命周期

**核心模块**

- 请求摘要
- `source_chain_id / target_chain_id`
- `request_type`
- `payload_hash`
- `nonce`
- `gateway_signature`
- `witness_signature_set`
- `ack_hash / ack_status`
- `final_status`
- 重试与补偿记录

### 4.3 跨链请求调试页

**页面目标**

- 研发与运维联查跨链请求问题

**核心模块**

- `request_id / ccr_id / ack_hash / tx_hash` 查询
- 请求原文/摘要切换
- ledger 状态
- 事件流轨迹

### 4.4 ack 与补偿联查页

**页面目标**

- 联查 ack 丢失、重复回执、补偿任务和人工介入记录

## 14. 数据本体存储与信任边界增量页面说明

### 14.0 数据原样处理与产品化加工跨平台继承要求

- 继承 `V1/V2` 的原始接入、元信息、加工任务和预览工件表达。
- 跨平台对象新增外部原始对象摘要、外部加工责任链摘要、外部质量证明和销毁承诺展示区。
- 页面必须区分本平台生成内容与合作平台声明内容，避免责任主体混淆。
- 跨平台信任边界页需新增外部存储命名空间、外部分层区域和 archive 承诺映射视图。

### 14.1 跨平台信任边界页

**页面目标**

- 展示合作伙伴、第三方可信存储、跨平台连接器和监管责任链对应的信任边界。

**关键字段**

- `partner_storage_capability`
- `partner_key_governance_capability`
- `partner_execution_boundary_capability`
- `cross_platform_trust_scope`
- `retention_obligation_snapshot`
- `external_destroy_attestation`

### 14.2 监管穿透页新增要求

- 监管查询必须能看到对象当前存储模式。
- 必须能展示哪一方可解密、哪一方可执行、何时销毁或保留。
- 必须支持导出跨平台信任证据摘要。

### 14.3 连接器详情页新增要求

- 展示连接器支持的平台托管、自持、受控执行能力。
- 展示对方的销毁/保留承诺和互认证据。

### 14.4 跨平台查询能力映射页

- 目标：查看本平台 `query_surface` 与合作平台外部查询能力、模板能力、输出边界和撤权回执之间的映射关系。
- 核心字段：`local_query_surface_id`、`partner_query_capability_id`、`partner_template_digest`、`output_boundary_snapshot`、`revoke_receipt`、`destroy_attestation`。
