## 4. 模型与算法页面

### 3.4 搜索增量要求

- 继承 V1 的数据商品、服务商品、卖方搜索
- 新增模型商品与模型服务搜索结果页
- 支持混合检索、同义词、自动补全和排序配置化
- 搜索结果仍需在 PostgreSQL 做最终业务可见性校验

### 3.5 元信息与数据契约继承要求

- 模型商品、受控计算商品、结果产品必须继承 V1 的元信息与数据契约页。
- 新增字段至少包括算法来源、执行环境摘要、结果导出边界、证明要求和回滚条件。

### 3.6 查询与执行面继承要求

- 继承 V1 的查询面与模板配置页、查询运行与结果记录页。

- 查询运行详情必须区分模板查询、clean room 查询、TVF 调用和模型结果查询四类执行模式。

## 14. 数据本体存储与信任边界增量页面说明

### 14.0 数据原样处理与产品化加工继承要求

- 继承 `V1` 的原始接入与加工页、元信息与数据契约页、SKU 配置页。
- 模型商品、训练任务和结果产品需要在上述页面中新增模型卡、执行环境、证明材料、结果摘要和导出边界区块。
- 预览工件从“样本/文本/图片预览”扩展为“模型卡/评估摘要/结果摘要/证明摘要”。
- 存储与信任策略页需要扩展到 `training / model / result / archive` 等分层区域。

### 14.1 模型与训练信任策略页

**页面目标**

- 为模型、算法、训练任务和计算任务配置存储边界、权重托管方式、训练数据驻留方式和结果导出边界。

**关键字段**

- `artifact_storage_mode`
- `weight_custody_mode`
- `training_data_residency_mode`
- `execution_visibility_level`
- `result_export_policy`
- `proof_binding_required`

### 14.2 模型详情页新增要求

- 展示模型是“文件交付”还是“服务调用”。
- 展示平台是否持有模型权重、是否仅代理服务权限。
- 展示训练来源和证明材料摘要。

### 14.3 计算任务与结果页新增要求

- 任务详情页必须显示数据是否出域、在哪个环境执行、谁实际执行。
- 结果页必须显示可导出范围、导出限制和证明绑定状态。

## 15. 支付、资金流与轻结算增量页面说明

### 15.1 自动打款批次页

- 目标：查看和执行自动打款、失败重试和批次回滚。
- 核心字段：`payout_instruction_id`、`batch_id`、`payout_mode`、`payout_status`、`provider_payout_no`。

### 15.2 渠道分账配置页

- 目标：维护渠道分账/子商户扩展能力与收益拆分绑定关系。
- 核心字段：`provider_account_id`、`sub_merchant_id`、`split_mode`、`settlement_rule_id`。

### 15.3 周期扣费联查页

- 目标：查看周期账单、扣费结果、失败原因和补扣计划。
- 核心字段：`billing_cycle`、`payment_intent_id`、`settlement_id`、`retry_count`。

### 15.4 跨境法币走廊扩展页

- 目标：在新加坡起步底座上配置新增法币走廊、伙伴能力和收款偏好增强。
- 核心字段：`corridor_policy_id`、`payer_jurisdiction_code`、`payee_jurisdiction_code`、`allowed_collection_currencies`、`allowed_payout_currencies`、`requires_manual_review`。

## 17. 审计、证据链与回放增量页面说明

### 17.1 训练/证明审计联查页

- 目标：按 `task_id`、`model_id`、`round_id`、`proof_id` 联查训练、证明和结果导出时间线。
- 核心字段：`task_id`、`proof_binding_required`、`environment_id`、`proof_artifact_ref`、`result_export_policy`。

### 17.2 任务回放页

- 目标：对训练、计算、联邦任务执行 dry-run 回放并输出状态差异与证据缺失。
- 核心字段：`replay_job_id`、`task_id`、`round_summary_digest`、`aggregate_digest`、`replay_status`。

## 18. 双层权威模型增量页面说明

### 18.1 训练 / 分润一致性联查页

- 目标：联查训练结果、证明状态、分润执行结果、外部回执与 Fabric 摘要状态。
- 核心字段：`proof_commit_state`、`external_fact_status`、`reconcile_status`、`reward_id`、`payout_instruction_id`。

## 19. 推荐增量页面说明

### 19.1 模型推荐运营页

- 目标：查看推荐模型注册、模型版本、模型效果指标和是否参与线上候选/排序。
- 核心字段：`model_registry_id`、`model_key`、`backend_type`、`model_registry_status`、`metrics_json`。

### 19.2 推荐实验页

- 目标：配置推荐实验、变体、流量比例和效果指标。
- 核心字段：`experiment_key`、`variant_key`、`traffic_ratio`、`assignment_rule`、`experiment_status`。

## 20. 日志、可观测性与告警增量页面说明

### 20.1 模型与任务可观测性页

- 目标：联查模型服务、训练任务、联邦任务、证明任务和分润任务的 trace、日志和告警。
- 核心字段：`trace_id`、`task_id`、`model_id`、`round_id`、`service_name`、`backend_key`、`alert_count`、`incident_ticket_id`。

### 20.2 公链与分润告警页

- 目标：查看凭证签发、锚定批次、分润结算、自动打款和支付回调扩展链路中的告警与工单。
- 核心字段：`alert_rule_id`、`alert_event_id`、`reward_id`、`payout_instruction_id`、`public_anchor_batch_id`、`alert_status`。

### 20.3 页面访问角色与关键权限补充

| 页面 | 主要访问角色 | 查看权限 | 关键操作权限 |
|---|---|---|---|
| 模型与任务可观测性页 | 可观测性运营员、节点运维管理员、Compute Environment Operator、证明服务管理员 | `ops.observability.read` / `ops.trace.read` | - |
| 公链与分润告警页 | 可观测性运营员、节点运维管理员、分润管理员、公链展示管理员 | `ops.alert.read` / `ops.incident.read` | `ops.alert.ack` / `ops.incident.manage` |

## 21. 敏感数据高阶治理页面

### 21.1 受控执行策略页

- 展示 `clean room / TEE / FL / MPC` 执行路径、执行边界、导出边界和证明要求。

### 21.2 远程证明与隐私预算页

- 展示远程证明、证明材料、差分隐私预算、预算消耗和预算告警。

### 21.3 销毁/保留证明页

- 展示到期对象、撤权对象、销毁证明、保留证明和例外审批记录。

## 22. 交易链监控、公平性与信任安全增量页面说明

### 22.1 任务与执行链监控页

- 目标：联查 `compute_task / training_task / compute_result / reward_record / payout_instruction` 在同一交易链上的状态。
- 核心字段：`task_id`、`environment_id`、`current_checkpoint_code`、`attestation_status`、`result_review_status`、`reward_record_id`、`external_fact_status`。

### 22.2 公平性事件与自动动作页

- 目标：查看任务超时、结果越权、分润/打款迟迟未确认等事件，并追踪自动阻断、冻结、补偿建议和人工处理结果。
- 核心字段：`fairness_incident_id`、`incident_type`、`auto_action_code`、`ticket_id`、`resolution_summary`、`closed_at`。
