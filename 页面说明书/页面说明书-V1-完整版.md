## 3. 全局页面规范

### 3.1 全局对象主键

- `org_id`
- `user_id`
- `did_id`
- `asset_id`
- `asset_version_id`
- `product_id`
- `order_id`
- `delivery_id`
- `case_id`
- `audit_id`
- `request_id`
- `tx_hash`

### 3.2 全局状态标签

- 商品：`draft / pending_review / listed / suspended / retired`
- 订单：`created / buyer_locked / seller_delivering / delivered / accepted / rejected / settled / closed / expired`
- 争议：`opened / evidence_collecting / auto_decision / manual_review / resolved / archived`
- 保证金：`pending_lock / locked / partially_penalized / released`

### 3.3 全局交互要求

- 所有写操作页面必须支持幂等提交，前端需附带 `Idempotency-Key`
- 所有敏感页面必须显示当前主体、角色、租户和作用域
- 所有关键动作页面必须显示审计留痕提示
- 下载类页面不得暴露真实对象路径
- 链相关页面必须展示 `request_id`、`tx_hash`、链状态和投影状态

本文件页面权限以以下文档为准：

- [权限点清单.md](/home/luna/Documents/DataB/权限设计/权限点清单.md)

- [菜单权限映射表.md](/home/luna/Documents/DataB/权限设计/菜单权限映射表.md)

页面实现要求：

- 每个页面至少定义 `查看权限`
- 每个主按钮必须绑定明确的 `执行权限`
- 高风险按钮必须标注“二次认证/人工确认/双人复核”要求

## 4. 门户与目录

### 4.1 首页

**页面目标**

- 提供平台导航、核心价值说明、热门行业入口和推荐商品入口

**目标角色**

- 游客
- 已登录买方
- 已登录卖方

**入口**

- 平台根路径
- 登录后默认落地页

**核心模块**

- 顶部导航
- 行业分类导航
- 热门商品区
- 最新上架区
- 平台能力说明
- 登录/注册入口

**关键展示字段**

- 商品标题
- 类别
- 标签
- 价格区间
- 商品类型
- 供方摘要

**状态与异常**

- 未登录时隐藏个性化内容
- 搜索服务异常时展示降级推荐和错误提示

### 4.2 行业专题页

**页面目标**

- 按行业展示专题商品、典型场景和模板化链路

**核心模块**

- 行业 Banner
- 场景说明
- 专题商品列表
- 合规提示
- 推荐模板说明

### 4.3 搜索页

**页面目标**

- 提供基于元数据和索引字段的商品搜索、筛选和排序能力

**核心模块**

- 关键词搜索框
- 精确筛选区
- 排序区
- 商品结果列表
- 搜索结果统计

**筛选字段**

- 商品类型
- 行业分类
- 标签
- 敏感等级
- 交付方式
- 价格模式
- 供方

**排序字段**

- 上架时间
- 价格
- 热度
- 信誉

**异常**

- 无结果提示
- 风险商品不展示购买入口

### 4.4 产品详情页

**页面目标**

- 展示单个商品的完整可售卖信息并承接下单

**核心模块**

- 商品基本信息
- 样本摘要
- 元数据详情
- 权利边界
- 合同/验收/退款/计费摘要
- 供方信息
- 合规与地域限制
- 审计/存证摘要
- 下单入口

**关键字段**

- `product_id`
- `asset_version_id`
- `full_hash`
- `sample_hash`
- `allowed_usage`
- `price`
- `delivery_type`
- `seller_org_id`
- `status`

**异常**

- `suspended / retired` 时禁止下单
- 缺少样本摘要时禁止上架态展示

## 15. 数据本体存储与信任边界增量页面说明

### 15.1 全局新增字段

以下字段在商品详情、上架、订单详情、交付页、审计联查页中必须统一展示或可配置：

- `storage_mode`
- `payload_location_type`
- `custody_mode`
- `key_control_mode`
- `platform_plaintext_access`
- `default_delivery_route`
- `retention_policy`
- `destroy_policy`
- `trust_evidence_count`

### 15.2 存储与信任策略配置页

**页面目标**

- 让供方为资产或版本配置存储模式、明文边界、密钥控制和生命周期策略。

**目标角色**

- 数据治理管理员
- 产品经理/上架管理员
- 安全管理员

**入口**

- 上架中心 -> 资产详情 -> 存储与信任策略

**核心模块**

- 存储模式选择
- payload 位置类型
- 平台明文接触开关
- 密钥控制模式
- 交付路由默认值
- 保留/销毁策略
- 信任证据上传区

**关键按钮**

- 保存策略
- 提交审核
- 上传信任证据

### 15.3 订单详情页新增信任边界卡片

必须新增展示：

- `storage_mode_snapshot`
- `delivery_route_snapshot`
- `trust_boundary_snapshot`
- 是否允许平台触达明文
- 当前交付执行方
- 当前令牌/授权有效期

### 15.4 文件/API/沙箱页新增要求

- 文件交付页必须显示对象是否为平台托管或卖方自持。
- API 开通页必须显示数据源位于平台还是卖方侧。
- 沙箱开通页必须显示“是否只允许结果导出、是否允许原始底表导出”。
- 审计联查页必须支持按存储模式、明文边界、销毁状态查询。

## 16. 支付、资金流与轻结算页面说明

### 16.1 支付中心

- 目标：统一查看 `payment_intent`、支付状态、回调状态和支付渠道。
- 核心字段：`payment_intent_id`、`provider_key`、`payment_method`、`amount`、`currency_code`、`payment_status`、`provider_reference_no`。

### 16.2 收费规则页

- 目标：维护平台服务费、渠道手续费、保证金规则和增值服务费参数。
- 核心字段：`fee_rule_id`、`fee_domain`、`calculation_method`、`parameter_json`、`effective_from`、`effective_to`。

### 16.3 对账中心

- 目标：导入渠道账单、展示 `reconciliation_statement` 与 `reconciliation_diff`。
- 核心字段：`statement_date`、`provider_key`、`import_status`、`diff_type`、`diff_status`。

### 16.4 Mock 支付调试页

- 目标：在不使用真实货币的前提下模拟支付、退款、打款和 webhook。
- 核心字段：`mock_payment_case_id`、`scenario_type`、`delay_seconds`、`duplicate_webhook`、`partial_refund_amount`。
