## 3. 全局页面规范

### 3.1 全局对象主键

- `org_id`
- `user_id`
- `did_id`
- `asset_id`
- `asset_version_id`
- `product_id`
- `order_id`
- `delivery_id`
- `case_id`
- `audit_id`
- `request_id`
- `tx_hash`

### 3.2 全局状态标签

- 商品：`draft / pending_review / listed / suspended / retired`
- 订单：`created / buyer_locked / seller_delivering / delivered / accepted / rejected / settled / closed / expired`
- 争议：`opened / evidence_collecting / auto_decision / manual_review / resolved / archived`
- 保证金：`pending_lock / locked / partially_penalized / released`

### 3.3 全局交互要求

- 所有写操作页面必须支持幂等提交，前端需附带 `Idempotency-Key`
- 所有敏感页面必须显示当前主体、角色、租户和作用域
- 所有关键动作页面必须显示审计留痕提示
- 下载类页面不得暴露真实对象路径
- 链相关页面必须展示 `request_id`、`tx_hash`、链状态和投影状态

本文件页面权限以以下文档为准：

- 每个页面至少定义 `查看权限`
- 每个主按钮必须绑定明确的 `执行权限`
- 高风险按钮必须标注“二次认证/人工确认/双人复核”要求

### 5.3 SKU 配置页

**页面目标**

- 为商品定义标准化可售 SKU

**支持 SKU**

- `FILE_STD` 文件快照
- `FILE_SUB` 版本订阅
- `SHARE_RO` 只读共享
- `API_SUB` API 订阅
- `API_PPU` API 按量
- `QRY_LITE` 模板查询 lite
- `SBX_STD` 查询沙箱
- `RPT_STD` 固定报告 / 结果产品

**关键规则**

- 必须先选择标准 SKU 真值，再填写商业套餐编码
- `sku_type` 只允许取 8 个标准 SKU 编码
- `sku_code` 表示商品内实现型套餐编码，必须唯一
- `trade_mode` 仅表示交付/计费路径，不替代 `sku_type`
- `SHARE_RO`、`QRY_LITE`、`SBX_STD` 必须拆开配置

**核心模块**

- 标准 SKU 类型切换
- 商业套餐编码输入
- 交易方式切换
- 价格模式配置
- 次数/期限配置
- 交付规则配置
- SLA 配置
- 默认争议规则展示

**关键输出**

- `sku_id`
- `sku_type`
- `sku_code`
- `trade_mode`
- `price_snapshot`
- `billing_rule`
- `acceptance_rule`
- `refund_rule`

### 5.4 模板绑定页

**页面目标**

- 绑定合同模板、验收模板、退款模板、计费模板

## 23. 交易链监控、公平性与信任安全页面说明

### 23.1 交易链监控总览页

- 目标：展示订单、交付、验收、结算、敏感执行、链上确认和监控告警的统一时间线。
- 核心字段：`order_id`、`request_id`、`trace_id`、`current_checkpoint_code`、`checkpoint_status`、`external_fact_status`、`proof_commit_state`、`reconcile_status`、`open_fairness_incident_count`。

### 23.2 生命周期检查点页

- 目标：查看某笔交易所有检查点的应到时间、已到时间、来源证据和是否超时。
- 核心字段：`trade_lifecycle_checkpoint_id`、`checkpoint_code`、`checkpoint_status`、`expected_by`、`occurred_at`、`source_type`、`source_ref_type`、`source_ref_id`。

### 23.3 外部事实回执页

- 目标：查看支付回调、打款回执、执行证明、销毁证明等外部事实的原始摘要与确认状态。
- 核心字段：`external_fact_receipt_id`、`fact_type`、`provider_type`、`provider_key`、`provider_reference`、`receipt_status`、`receipt_hash`、`confirmed_at`。

### 23.4 公平性事件页

- 目标：查看迟交付、恶意拒收、回执缺失、越权导出等公平性事件，并联动风控与工单。
- 核心字段：`fairness_incident_id`、`incident_type`、`severity`、`lifecycle_stage`、`fairness_incident_status`、`auto_action_code`、`assigned_role_key`、`closed_at`。
