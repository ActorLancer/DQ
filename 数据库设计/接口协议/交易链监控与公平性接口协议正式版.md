# 交易链监控与公平性接口协议正式版

## 1. 文档目标

本协议用于统一平台在 `V1 / V2 / V3` 的以下接口语义：

- 交易链总览
- 生命周期检查点查询
- 外部事实回执查询与确认
- 公平性事件查询与处理
- 监控策略查询与修改
- 链投影缺口查询与关闭
- 任务交易链联查
- 跨链 / 跨平台交易链联查

## 2. 协议原则

- 交易链监控接口是“控制面 + 联查面”，不是主状态写入面。
- 数据库业务状态继续是业务运营主状态机。
- Fabric 继续是可信确认层，不被监控接口替代。
- 外部事实回执必须独立建模，不允许以备注或日志文本代替。
- 公平性事件只能处理，不得通过接口直接篡改业务主状态。
- 高风险动作必须带 `X-Step-Up-Token` 并再次进入审计链。

## 3. 公共请求头

## 6. V1 接口

### 6.1 交易链总览

`GET /api/v1/ops/trade-monitor/orders/{orderId}`

鉴权：

- `ops.trade_monitor.read`

返回：

- `TradeMonitorOverview`
- 最近检查点摘要
- 最近外部事实摘要
- 最近公平性事件摘要
- 最近链投影缺口摘要

### 6.2 生命周期检查点查询

`GET /api/v1/ops/trade-monitor/orders/{orderId}/checkpoints`

过滤建议：

- `checkpoint_code`
- `checkpoint_status`
- `lifecycle_stage`
- `from`
- `to`

### 6.3 外部事实回执查询

`GET /api/v1/ops/external-facts`

鉴权：

- `ops.external_fact.read`

过滤建议：

- `order_id`
- `ref_type`
- `ref_id`
- `fact_type`
- `provider_type`
- `receipt_status`
- `request_id`
- `trace_id`
- `from`
- `to`

### 6.4 外部事实回执确认

`POST /api/v1/ops/external-facts/{id}/confirm`

鉴权：

- `ops.external_fact.manage`

请求建议字段：

- `confirm_result`
- `reason`
- `operator_note`

要求：

- 必须 `step-up`
- 必须审计
- 不得直接改写业务主状态，只能写回执确认结果并触发后续规则评估

### 6.5 公平性事件查询

`GET /api/v1/ops/fairness-incidents`

鉴权：

- `risk.fairness_incident.read`

过滤建议：

- `order_id`
- `incident_type`
- `severity`
- `fairness_incident_status`
- `assigned_role_key`
- `assigned_user_id`
- `request_id`
- `trace_id`

### 6.6 公平性事件处理

`POST /api/v1/ops/fairness-incidents/{id}/handle`

鉴权：

- `risk.fairness_incident.handle`

请求建议字段：

- `action`
- `resolution_summary`
- `auto_action_override`
- `freeze_settlement`
- `freeze_delivery`
- `create_dispute_suggestion`

要求：

- 必须 `step-up`
- 必须审计
- 只能处理事件与联动动作，不得直接篡改业务事实

### 6.7 监控策略查询

`GET /api/v1/ops/monitor-policies`

- 必须 `step-up`
- 必须审计

## 7. V2 增量接口

### 7.1 任务交易链联查

`GET /api/v2/ops/trade-monitor/tasks/{taskId}`

鉴权：

- `ops.trade_monitor.read`

返回建议补充：

- `compute_task` / `training_task` 基本状态
- 执行环境摘要
- 执行证明摘要
- 结果披露审查状态
- 分润 / 打款状态

### 7.2 V2 增量规则

- `task_id / model_id / reward_id / payout_instruction_id` 必须成为正式联查主键。
- 任务完成、结果生成、分润完成、打款完成必须分别形成独立检查点，不允许合并成单一“已完成”。

## 8. V3 增量接口

### 8.1 跨链 / 跨平台交易链联查

`GET /api/v3/ops/trade-monitor/crosschain/{ccrId}`

鉴权：

- `ops.trade_monitor.read`

返回建议补充：

- `ccr_id`
- `ack_hash`
- `partner_id`
- `compensation_status`
- `target_ack_status`
- `crypto_transfer_id`
- `cross_platform_external_facts`

### 8.2 V3 增量规则

- `ccr_id / ack_hash / partner_id / crypto_transfer_id / regulator_query_id` 必须成为正式联查主键。
- 对端 ack、伙伴回执、跨平台结算确认都只能作为 `ExternalFactReceipt` 或等价对象进入统一监控模型。

## 9. 事件联动要求

以下动作必须同步写入审计链：

- 回执确认
- 公平性事件处理
- 监控策略修改
- 投影缺口关闭
- 任何由监控触发的冻结、阻断、补偿建议、争议建议

## 10. 错误码建议

- `TRADE_MONITOR_SCOPE_FORBIDDEN`
- `TRADE_MONITOR_ORDER_NOT_FOUND`
- `TRADE_CHECKPOINT_NOT_FOUND`
- `EXTERNAL_FACT_NOT_FOUND`
- `EXTERNAL_FACT_CONFIRM_REQUIRES_STEP_UP`
- `EXTERNAL_FACT_CONFLICT_DETECTED`
- `FAIRNESS_INCIDENT_NOT_FOUND`
- `FAIRNESS_INCIDENT_ALREADY_CLOSED`
- `FAIRNESS_ACTION_FORBIDDEN`
- `MONITOR_POLICY_NOT_FOUND`
- `MONITOR_POLICY_CHANGE_FORBIDDEN`
- `PROJECTION_GAP_NOT_FOUND`
- `PROJECTION_GAP_RESOLVE_FORBIDDEN`
- `PROJECTION_GAP_DRY_RUN_REQUIRED`

## 11. 一句话结论

交易链监控接口的目标不是替代业务主状态机，而是把完整交易链的检查点、外部事实、公平性事件和链投影缺口统一拉到一个正式控制面里，供监控、风控、审计和运维共同使用。
