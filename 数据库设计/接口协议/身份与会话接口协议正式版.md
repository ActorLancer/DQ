# 身份与会话接口协议正式版

## 1. 文档定位

本协议用于统一平台在以下接口上的对外语义：

- 主体注册
- 登录与登出
- `MFA`
- 会话与设备
- 企业 `OIDC/SAML/SCIM`
- `step-up`
- Fabric 身份与证书治理

## 2. 协议原则

- 浏览器门户：推荐服务端会话 + `HttpOnly Cookie`
- API / SDK：短期 access token + refresh token
- 企业联邦：`OIDC` 优先，`SAML`/`SCIM` 为后续扩展
- Fabric 证书不直接等于普通网页登录凭证
- 所有高风险接口必须支持 step-up 与审计

## 3. 公共请求头

- `X-Request-Id`
- `X-Idempotency-Key`：适用于写接口
- `Authorization: Bearer <access_token>`：API/SDK 接口
- `X-Step-Up-Token`：高风险动作二次认证后下发的短时令牌

## 4. V1 接口

### 4.1 注册与登录

- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`
- `POST /api/v1/auth/token/refresh`
- `POST /api/v1/auth/logout`
- `GET /api/v1/auth/me`

### 4.2 MFA / 设备 / 会话

- `GET /api/v1/iam/mfa/authenticators`
- `POST /api/v1/iam/mfa/authenticators`
- `DELETE /api/v1/iam/mfa/authenticators/{id}`
- `GET /api/v1/iam/devices`
- `POST /api/v1/iam/devices/{id}/revoke`
- `GET /api/v1/iam/sessions`
- `POST /api/v1/iam/sessions/{id}/revoke`
- `POST /api/v1/iam/step-up/challenges`
- `POST /api/v1/iam/step-up/challenges/{id}/verify`

### 4.3 成员邀请与企业 OIDC

- `POST /api/v1/iam/invitations`
- `GET /api/v1/iam/invitations`
- `POST /api/v1/iam/invitations/{id}/cancel`
- `POST /api/v1/iam/sso/connections`
- `PATCH /api/v1/iam/sso/connections/{id}`
- `GET /api/v1/iam/sso/connections`

### 4.4 Fabric 身份与证书

- `GET /api/v1/iam/fabric-registries`
- `POST /api/v1/iam/fabric-registries`
- `GET /api/v1/iam/fabric-identities`
- `POST /api/v1/iam/fabric-identities/{id}/issue`
- `POST /api/v1/iam/fabric-identities/{id}/revoke`
- `GET /api/v1/iam/certificates`
- `POST /api/v1/iam/certificates/{id}/revoke`

## 5. V2 接口增量

- `GET /api/v2/iam/provisioning/jobs`

## 6. V3 接口增量

- `POST /api/v3/iam/external-identities`
- `GET /api/v3/iam/external-identities`
- `POST /api/v3/iam/risk-auth/policies`
- `PATCH /api/v3/iam/risk-auth/policies/{id}`

## 7. 错误码建议

- `AUTH_INVALID_CREDENTIAL`
- `AUTH_MFA_REQUIRED`
- `AUTH_SESSION_REVOKED`
- `AUTH_DEVICE_REVOKED`
- `AUTH_STEP_UP_REQUIRED`
- `AUTH_STEP_UP_EXPIRED`
- `AUTH_SSO_CONNECTION_INVALID`
- `AUTH_FABRIC_IDENTITY_NOT_BOUND`
- `AUTH_CERTIFICATE_REVOKED`
- `AUTH_CERTIFICATE_EXPIRED`

## 8. 一句话结论

身份与会话接口必须把“人、设备、会话、外部身份、链上证书、高风险二次认证”统一编排起来。后续正式技术方案和开发文档应以这份协议为接口语义基线。
