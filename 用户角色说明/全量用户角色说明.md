# 数据交易平台全量用户角色说明

## 1. 文档说明

本文件统一覆盖以下输入：

- `正式PRD/正式PRD-V1-完整版.md`
- `正式PRD/正式PRD-V2-完整版.md`
- `正式PRD/正式PRD-V3-完整版.md`
- `业务流程/业务流程图-V1-完整版.md`
- `业务流程/业务流程图-V2-完整版.md`
- `业务流程/业务流程图-V3-完整版.md`
- `原始PRD/角色与权限矩阵详细稿.md`

目标是形成一份贯穿 `V1 / V2 / V3` 的长期可用角色说明。
原则是：

- `V1` 可以先只启用最小角色子集
- 但角色体系、身份层级、职责分离、机器身份和监管/生态角色必须一次性定义完整
- 避免 V2/V3 因新增模型、联邦、跨链、图风控、监管协同而推翻原权限体系

## 2. 角色体系设计原则

### 2.1 基本原则

- 采用 `RBAC + ABAC + PBAC + Scope` 的混合模型
- 权限授予对象不仅包括人，也包括应用、连接器、执行环境、网关、见证人和监管观察身份
- 角色只是基础许可，最终放行还要叠加：
  - 主体状态
  - 作用域
  - 商品/SKU 类型
  - 合同与 Usage Policy
  - 环境策略
  - 合规/风控结果

### 2.2 设计目标

- 支持平台运营、租户交易、机器执行、监管协同、生态互联五大角色域
- 支持阶段性启用，不要求 V1 一次实现全部复杂审批与治理链路
- 保持职责分离，避免单一角色闭环完成高风险动作

### 2.3 角色启用策略

```text
V1:
- 启用平台运营与审核主角色
- 启用租户基础交易角色
- 启用开发者/应用管理员
- 启用一致性运营角色
- 启用基础机器身份
- 监管与高级生态角色先以只读/预留为主

V2:
- 正式启用算法、执行环境、联邦、证明、公链、分润相关角色

V3:
- 正式启用跨链、见证、图风控、监管协同、连接器互联、生态合作角色
```

## 3. 身份层级

## 3.1 组织身份

用于合同、账单、责任归属、合作关系与监管归属。

组织身份类型包括：

- 企业
- 政府/公共机构
- 科研/事业单位
- 服务商
- 境外主体
- 合作伙伴主体
- 监管主体

## 3.2 人工身份

用于登录、人工操作、审核、裁决、治理与留痕。

人工身份包括：

- 平台人员
- 租户管理员
- 业务操作用户
- 审核/法务/财务/安全人员
- 监管观察员
- 合作伙伴人工操作员

## 3.3 服务身份

用于 API 调用、自动化任务、连接器、同步与证明服务。

服务身份包括：

- 应用账号
- API Client
- 连接器服务账号
- 预言机/同步服务账号
- 批处理任务账号
- 公链锚定服务账号
- 凭证签发服务账号

## 3.4 链上身份

用于联盟链成员准入、交易存证、公链展示和监管节点接入。

链上身份包括：

- 联盟链节点组织身份
- 链上签名身份
- 公链凭证签发身份
- 监管观察节点身份
- 跨链网关签名身份
- 见证人签名身份

## 4. 角色总览

角色体系建议分为 6 层：

1. 平台侧角色
2. 租户侧角色
3. 机器与环境角色
4. 监管与审计角色
5. 跨链与生态角色
6. 开发与调试角色

## 5. 平台侧角色

## 5.1 平台超级管理员

**定位**

- 平台最高管理角色

**核心职责**

- 系统级配置
- 关键策略开关
- 关键冻结/解冻确认
- 节点、链、密钥、环境高权限管理

**风险边界**

- 不应直接处理日常审核、争议裁决和普通账单执行

**阶段**

- `V1` 启用

## 5.2 运营管理员

**职责**

- 门户内容维护
- 行业专题与商品运营
- 工单协调
- 用户支持协调

**边界**

- 不做最终合规放行
- 不做高风险冻结

## 5.3 主体审核员

**职责**

- 组织准入审核
- 认证资料核验
- 主体状态变更建议

**边界**

- 不负责商品审核和财务结算

## 5.4 产品审核员

**职责**

- 商品、SKU、样例、元数据审核
- 上架驳回和下架建议

**边界**

- 不负责合规规则最终解释

## 5.5 合规审核员

**职责**

- 高风险规则审查
- 用途、地域、导出限制、敏感等级审查
- 增强审批支持

**阶段**

- `V1` 启用
- `V2` 处理受控训练、受控输出、证明要求
- `V3` 处理跨域互认与监管协同规则

## 5.6 风控专员

**职责**

- 风险命中处置
- 黑名单、冻结建议、风险升级
- 风险工单跟进

**阶段**

- `V1` 规则风控
- `V2` 任务风控、分润风控
- `V3` 图风控处置

## 5.7 法务专员

**职责**

- 合同模板审核
- 条款变更支持
- 争议责任判定支持

## 5.8 财务结算专员

**职责**

- 账单确认
- 退款、赔付执行
- 人工结算
- 半自动分账确认

**阶段**

- `V1` 核心启用
- `V2` 与自动分润联动复核

## 5.9 争议处理专员

**职责**

- 争议受理
- 证据归集
- 人工复核流转
- 裁决结果归档

## 5.10 审计管理员

**职责**

- 证据联查
- 证据包导出
- 审计归档
- 审计只读支持

**边界**

- 不具备业务放行权限

## 5.11 安全运营管理员

**职责**

- 密钥、证书、令牌、异常封禁
- 安全事件处置
- 高风险服务身份吊销

## 5.12 搜索运营员

**职责**

- 搜索索引同步联查
- Reindex 与别名切换
- 排序配置治理
- 搜索缓存失效与恢复

**边界**

- 不直接修改商品主数据事实
- 不绕过 PostgreSQL 最终业务校验
- 不替代审核、合规和风控决策

## 5.13 节点运维管理员

**职责**

- 联盟链节点、链适配器、同步器、网关健康管理
- 生产环境变更和故障恢复

## 5.14 公链展示管理员

**职责**

- 公链锚定任务管理
- 凭证签发/吊销
- 公开验证页内容控制

**阶段**

- `V1` 可演示级预留
- `V2` 正式启用

## 5.14 分润管理员

**职责**

- 分润规则配置
- 贡献度复核
- 分润异常处理

**阶段**

- `V2` 启用

## 5.15 图风控运营角色

**职责**

- 图谱分析结果研判
- 风险告警闭环
- 转工单、冻结建议、恢复观察

**阶段**

- `V3` 启用

## 5.16 监管协同管理员

**职责**

- 监管查询权限配置
- 监管材料导出协调
- 监管指令执行回传

**阶段**

- `V3` 启用

## 5.17 生态合作管理员

**职责**

- 连接器接入
- 合作伙伴能力开放
- 数据空间互认关系维护

**阶段**

- `V3` 启用

## 6. 租户侧角色

## 6.1 Tenant 管理员

**职责**

- 用户、部门、应用管理
- 角色分配
- 组织级配置

**阶段**

- `V1` 启用

## 6.2 Department 审批人

**职责**

- 部门级审批、预算确认、扩权确认、敏感操作确认

**阶段**

- `V1` 可保留对象与作用域
- `V2/V3` 完整启用

## 6.3 数据治理管理员

**职责**

- 产品、SKU、样例、元数据维护
- 产品版本升级

## 6.4 产品经理 / 上架管理员

**职责**

- 创建商品
- 配置价格、描述、模板、标签
- 提交审核

## 6.5 采购经理

**职责**

- 选品
- 下单
- 跟踪订单与账单
- 发起续费、退款、扩权

## 6.6 法务经理

**职责**

- 审阅合同
- 发起条款确认
- 法律风险复核

## 6.7 财务经理

**职责**

- 预算确认
- 查看账单
- 发票与结算对接

## 6.8 开发者 / 应用管理员

**职责**

- 测试应用管理
- API Key 管理
- 调用调试
- 应用绑定

**阶段**

- `V1` 启用
- `V2` 扩展到算法上传、任务调试、凭证联查、分润联查
- `V3` 扩展到跨链调试、图风控演练、监管演练环境

## 6.9 安全管理员

**职责**

- 组织内应用安全、白名单、访问限制、密钥配合管理

## 6.10 审计只读角色

**职责**

- 只读查看审计记录、账单、订单、争议与证据摘要

## 6.11 业务分析师 / 查询使用者

**职责**

- 使用已购买的数据/API/沙箱/报告
- 查看使用记录

## 6.12 报告接收者

**职责**

- 只接收报告成果
- 不具备商品配置和账单操作权限

## 6.13 模型使用者

**职责**

- 调用模型 API
- 查看模型结果
- 发起续费或输出权限申请

**阶段**

- `V2` 启用

## 6.14 联邦任务发起方

**职责**

- 创建联邦任务
- 定义参与条件、奖励规则、输出边界

**阶段**

- `V2` 启用

## 6.15 联邦参与方管理员

**职责**

- 报名联邦任务
- 维护节点状态
- 查看贡献记录和奖励结果

**阶段**

- `V2` 启用

## 6.16 结果接收者

**职责**

- 接收受控计算、MPC 或证明类结果

**阶段**

- `V2` 启用

## 7. 机器与环境角色

## 7.1 Application

**职责**

- 代表租户下应用进行 API 调用或自动化操作

**边界**

- 不得绕过 `Usage Policy`

## 7.2 API Consumer App

**职责**

- 调用数据 API、模型 API、公开验证 API

- `V3` 正式成为互联主体

## 7.4 Sandbox Operator

**职责**

- 管理查询沙箱、开通席位、控制会话与导出限制

## 7.5 Compute Environment Operator

**职责**

- 管理受控计算环境
- 控制镜像、网络访问、输出边界、运行状态

**阶段**

- `V2` 启用

## 7.6 联邦协调器服务

**职责**

- 任务编排
- 节点招募协调
- 轮次状态推进

**阶段**

- `V2` 启用

## 7.7 聚合器服务

**职责**

- 模型聚合
- 轮次结果摘要固化

**阶段**

- `V2` 启用

## 7.8 证明服务管理员 / 服务身份

**职责**

**阶段**

- `V2` 启用

## 14. 职责分离规则

以下动作不应由单一人工角色闭环完成：

- 主体准入审核 与 主体冻结解除
- 商品上架提交 与 商品最终审核
- 合规放行 与 风险放行
- 账单生成 与 退款/赔付最终执行
- 争议受理 与 最终裁决
- 分润规则配置 与 分润复核
- 监管查询配置 与 审计证据归档删除类操作
- 跨链请求发起配置 与 见证签名管理
- 连接器接入审批 与 连接器健康状态伪造

### 15.1 V1 高风险动作

- 退款/赔付执行
- 密钥/证书吊销

建议控制：

- 角色权限 + 二次认证
- 必要时管理员确认

### 15.2 V2 高风险动作

- 算法白名单放行
- 受控训练放行
- 结果级导出放行
- 分润执行与回滚

建议控制：

## 16. 权限边界提醒

- 商品购买权限不等于算法执行权限
- 结果访问权限不等于原始数据访问权限
- 公链凭证签发权限不等于交易放行权限
- 审计查看权限不等于业务修改权限
- 监管穿透权限不等于平台运营权限
- 连接器接入权限不等于跨链放行权限

## 18. 一句话结论

这套角色模型的核心不是“把所有角色立刻都做成复杂审批流”，而是：

- 现在就把角色层级、职责边界、机器身份、监管身份和生态角色定完整
- `V1` 只启用最小可运行子集
- `V2/V3` 在既有角色骨架上扩展，不重构权限体系

## 19. 存储与信任边界相关角色补充

### 19.1 平台侧新增角色

#### 数据托管管理员

职责：

## 22. 审计、证据链与回放角色补充

### 22.1 平台侧

- `审计管理员`
  - 扩展负责审计联查、证据包导出、回放执行、保留策略、`legal hold` 和锚定批次治理。
- `安全运营管理员`
  - 仅在异常调查或 `break-glass` 场景下查看敏感原文和访问记录。
- `对账专员 / 财务结算专员`
  - 只查看支付、退款、赔付、打款、对账相关审计对象，不默认看全站原文。

### 22.2 租户侧与监管侧

- `审计只读角色`
  - 只允许联查和脱敏视图，不允许导出原文或执行回放。
- `监管观察员`
  - 默认只读摘要/脱敏视图，监管导出需额外授权。

### 22.3 角色边界原则

- 审计查看权限与业务查看权限分离
- 原文查看、导出、回放、解除 hold 必须二次认证
- 审计查看者本身也必须进入审计链

## 23. 双层权威模型与一致性角色补充

### 23.1 平台侧

- `一致性运营员`
  - 负责 `outbox_event`、Kafka 发布、`dead_letter_event`、重处理、对账修复与一致性联查。
- `节点运维管理员`
  - 负责 Fabric writer、链适配器、消息链路、消费者健康和失败恢复。
- `审计管理员`
  - 负责一致性修复过程中的证据链联查和回放授权，但不直接替代业务放行。
- `搜索运营员`
  - 负责商品、卖方、模型搜索索引同步、别名切换、缓存失效和排序配置治理。

### 23.2 角色边界原则

- 一致性运营员可以处理事件流和死信，但不能越权修改业务对象事实。
- 搜索运营员可以重建搜索读模型，但不能越权修改商品、卖方、价格和信誉主数据。
- 财务、支付、证书、跨链等外部事实对象的最终成立，仍以来源回执为准。
- Kafka 运维权限与业务操作权限必须分离。
