<th><p>function acceptOrder(orderId, buyerSignature):</p>
<p>order = getOrderProjection(orderId)</p>
<p>assert order.status == "delivered"</p>
<p>assert verifyBuyerSignature(order.buyerDid, buyerSignature)</p>
<p>chainTx = OrderEscrow.acceptOrder(orderId, hash(buyerSignature))</p>
<p>waitForEvent("OrderAccepted", orderId, chainTx)</p>
<p>projectOrderStatus(orderId, "accepted")</p>
<p>SettlementService.settle(orderId)</p>
<p>ReputationService.stageUpdate(orderId)</p>
<p>AuditService.record("ORDER_ACCEPTED", orderId, chainTx)</p></th>

<table>
<colgroup>
<col style="width: 100%" />
</colgroup>
<thead>
<tr class="header">
<th><p>function autoResolve(caseId):</p>
<p>caseObj = getCase(caseId)</p>
<p>order = getOrder(caseObj.orderId)</p>
<p>if order.deliveredAt &gt; order.sellerDeadline:</p>
<p>return sellerLoses("TIMEOUT")</p>
<p>if caseObj.reason == "HASH_MISMATCH" and verifyHashMismatch(order) == true:</p>
<p>return sellerLoses("HASH_MISMATCH")</p>
<p>if caseObj.reason == "BUYER_REJECT" and verifyHashMatch(order) == true and receiptExists(order):</p>
<p>return buyerLoses("MALICIOUS_REJECT")</p>
<p>return escalateToManual(caseId)</p></th>

## 16.3 术语表

**术语表**

| **术语**                 | **解释**                                                   |
|--------------------------|------------------------------------------------------------|
| 数据资产                 | 被业务和治理规则识别、可描述、可授权、可交易的数据对象。   |
| 商品 Product             | 面向市场展示和交易的数据资产封装。                         |
| 保证金 Deposit           | 为了约束违约和恶意行为而锁定的经济担保。                   |
| 交付承诺 Delivery Commit | 卖方关于具体交付对象的链上摘要承诺。                       |
| 联邦学习                 | 原始数据不集中，仅交换模型更新或梯度的分布式协同学习方式。 |
| 预言机 Oracle            | 把链下事实转化为链上可用信息的可信中间层。                 |
| 图风控                   | 利用关系图和图算法识别复杂关联风险的方法。                 |

- 莫梓嘉，赵晨. 融合区块链和联邦学习技术的数据资产流通交易方案研究. 大数据, 2025.

- 本设计文档在上述论文提出的关键思想基础上进行了工程化补全，补充了统一对象模型、三阶段实施策略、数据库设计、接口设计、部署与运维设计、可观测性和风控治理方案。
