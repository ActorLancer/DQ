# 5. 统一数据对象模型与主数据规范

## 5.1 核心对象清单

**核心对象**

| **对象**                   | **主键**         | **说明**                           |
|----------------------------|------------------|------------------------------------|
| 组织 Organization          | org_id           | 企业、机构、监管或服务主体。       |
| 用户 User                  | user_id          | 组织下的操作用户或系统账户。       |
| 数字身份 DID/Cert          | did_id / cert_sn | 组织和用户的链上身份映射。         |
| 数据资产 DataAsset         | asset_id         | 逻辑上的可交易数据资产。           |
| 数据版本 DataAssetVersion  | asset_version_id | 某一批次、某个快照或某个模式版本。 |
| 数据商品 Product           | product_id       | 面向交易市场曝光的商品封装。       |
| 订单 Order                 | order_id         | 买卖双方围绕商品形成的交易实例。   |
| 交付 Delivery              | delivery_id      | 链下交付对象与交付回执。           |
| 争议 Case                  | case_id          | 围绕订单的申诉、证据和裁决。       |
| 信誉 Snapshot              | reputation_id    | 组织、用户或节点的信誉快照。       |
| 训练任务 TrainingTask      | task_id          | 二期引入的联邦学习或协同任务。     |
| 跨链请求 CrossChainRequest | ccr_id           | 三期引入的跨域消息与回执对象。     |
| 审计事件 AuditEvent        | audit_id         | 全系统统一的审计记录。             |

## 5.3 统一生命周期

**对象生命周期（摘要）**

| **对象**          | **主要状态**                                                                                                         | **关键触发器**                           |
|-------------------|----------------------------------------------------------------------------------------------------------------------|------------------------------------------|
| Product           | draft -\> pending_review -\> listed -\> suspended -\> retired                                                        | 提交审核、审核通过、违规下架、到期退市   |
| Order             | created -\> buyer_locked -\> seller_delivering -\> delivered -\> accepted/rejected -\> settled/arbitrated -\> closed | 下单、托管、交付、验真、仲裁、结算       |
| Case              | opened -\> evidence_collecting -\> auto_decision/manual_review -\> resolved -\> archived                             | 申诉、证据提交、裁决、归档               |
| TrainingTask      | draft -\> recruiting -\> running -\> aggregating -\> rewarded -\> completed/aborted                                  | 任务创建、节点加入、轮次训练、奖励、结束 |
| CrossChainRequest | initiated -\> signed -\> relayed -\> acknowledged -\> finalized/failed                                               | 发起、签名、跨链转发、回执、完成         |
