# 15. 测试策略、验收标准与实施里程碑

## 15.1 测试策略

**测试分层**

| **测试类型** | **范围**                                         | **重点**                 |
|--------------|--------------------------------------------------|--------------------------|
| 单元测试     | 服务方法、风控规则、公式计算、合约函数           | 保证逻辑正确性。         |
| 集成测试     | 链上链下协作、对象存储、消息总线、数据库投影     | 验证状态闭环。           |
| 安全测试     | 认证授权、越权、令牌泄露、对象越权下载、合约攻击 | 验证安全边界。           |
| 性能测试     | 目录查询、下单、交付、验真、事件投影、训练任务   | 确定容量瓶颈。           |
| 容灾测试     | 节点故障、数据库主备切换、消息总线异常、证书轮换 | 验证恢复能力。           |
| UAT          | 真实用户流程演练                                 | 验证可用性与流程合理性。 |

## 15.2 分阶段验收标准

**验收标准概览**

| **阶段** | **必须通过项**                             | **通过判定**                                  |
|----------|--------------------------------------------|-----------------------------------------------|
| Phase 1  | 交易闭环、仲裁闭环、评分闭环、审计闭环     | 连续演示 20 笔以上订单无关键错误。            |
| Phase 2  | 训练任务闭环、策略控制、奖励闭环、模型注册 | 至少 3 个节点完成多轮训练并有可解释奖励结果。 |
| Phase 3  | 跨链闭环、图风控告警、监管穿透、冻结处置   | 至少 1 条跨链链路成功、1 类风险识别成功。     |

## 15.3 实施里程碑建议

**实施计划（示例）**

| **阶段**       | **周期建议** | **主要产出**                             |
|----------------|--------------|------------------------------------------|
| 需求与设计冻结 | 2~4 周       | 领域模型、接口草案、链选型、PoC 结论。   |
| Phase 1 研发   | 8~12 周      | MVP 系统、合约、对象存储、基础运维。     |
| Phase 1 稳定化 | 4~6 周       | 性能优化、运维补齐、灰度上线。           |
| Phase 2 研发   | 10~16 周     | 联邦任务、策略引擎、贡献激励、模型资产。 |
| Phase 3 研发   | 12~20 周     | 跨链网关、预言机、图风控、监管视图。     |

| **实施建议：**不要一开始就并行做三期。最稳妥的方式是：先把 Phase 1 做到能够稳定运营，再把高敏感数据场景迁移到 Phase 2，最后在真实业务压力和治理需求驱动下建设 Phase 3。 |
